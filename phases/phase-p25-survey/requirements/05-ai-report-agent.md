Use Case 名称：
使用智能 Agent 生成和改写问卷报告

Actor：
Survey Owner、Survey Admin

目标：
用户在查看问卷结果时，通过智能 Report Agent 生成报告、预览报告结构、追问数据含义、改写报告口径，并把报告输出为团队可使用的文档。

系统边界：
Survey / Survey Report Agent

前端入口：
- Survey 结果页 > Report > Generate with AI。
- Survey 结果页 > Report Agent 对话面板。
- Survey 结果页 > Preview Report Outline。

前置条件：
- 用户已登录。
- 用户拥有目标问卷的结果查看权限。
- 目标问卷存在。
- 问卷有可用于分析的结构；可以没有答卷，但没有答卷时只能生成报告框架，不能生成真实结论。
- 至少有一个可用 LLM 模型配置。

触发条件：
用户点击 Preview Report Outline、Generate with AI、Regenerate，或在 Report Agent 中提出问题，例如“帮我总结这次商品反馈的主要问题”。

主流程：
1. 用户在问卷结果页进入 Report。
2. 系统展示报告状态：未生成、已生成、生成中或生成失败。
3. 用户点击 Preview Report Outline。
4. Agent 基于问卷题目、分类、题型和报告目标生成报告框架预览，包括封面摘要、样本概览、核心指标、分类分析、开放反馈主题、风险与建议、附录和方法边界。
5. 用户可以编辑报告框架的章节标题、章节顺序、关注重点和报告语气，例如“面向老板”“面向产品经理”“更短一点”“突出价格问题”。
6. 用户点击 Generate with AI。
7. 系统把当前筛选范围、有效答卷、问卷结构、报告框架和用户偏好提交给 Agent。
8. Agent 生成报告，报告必须展示样本量、数据范围、生成时间、模型名称、关键结论、证据引用、建议行动和不确定性说明。
9. 用户可以在报告旁继续提问，例如“哪些反馈最影响复购”“把开放题按主题聚类”“给我 5 条产品改进建议”“生成一版给销售团队看的摘要”。
10. Agent 以对话方式回答，并可以提出报告修改建议；用户确认后，修改写入报告文档。
11. 用户可以要求 Agent 重新组织报告章节、压缩报告、展开某个维度、改变语气、生成中英文版本或生成会议汇报版。
12. 用户点击 Export PDF、Export Word 或保存为团队报告文档，把报告输出给团队。

备选流程：
- A1：没有答卷时，Agent 只生成报告框架、预期指标和“需要回收的数据”清单，不生成伪结论。
- A2：用户切换筛选条件后，系统提示报告数据范围已变化，用户可选择基于新范围 Regenerate。
- A3：用户选择“保守模式”，Agent 只输出数据中有明确支持的结论。
- A4：用户选择“洞察模式”，Agent 可以输出假设和建议，但必须标注依据和不确定性。
- A5：用户选择报告模板，例如商品反馈、NPS 满意度、活动报名复盘、课程评价、团队复盘。

异常流程：
- E1：模型调用失败，Report Agent 显示失败原因、Retry 和 Switch model；旧报告不被覆盖。
- E2：答卷数据为空或样本过少，报告生成区域显示样本不足提示，并要求用户确认是否生成框架报告。
- E3：报告生成过程中用户离开页面，系统保留生成任务状态；返回时可查看完成、失败或继续等待。
- E4：Agent 生成内容缺少样本量、数据范围或方法边界，系统标记报告不完整并要求重新生成或补充。
- E5：用户无导出权限时，导出入口隐藏或禁用，Report Agent 仍可在允许范围内展示摘要。

权限与可见性：
- Survey Owner 可以对自己问卷生成、改写、导出和发送报告。
- Survey Admin 可以对权限范围内问卷生成、改写、导出和发送报告。
- Survey Member 不自动拥有报告生成或原始数据访问权限。
- Respondent 不能访问 Report Agent。
- Visitor/未加入团队的用户不能访问报告或报告 Agent。

后置条件：
- 用户获得一份可编辑、可导出、可发送的报告文档。
- 报告保留生成时间、数据范围、模型名称、用户选择的报告框架和方法边界。
- 如果用户确认 Agent 修改建议，报告内容更新并保留最新版本。
- 失败时，旧报告和用户选择的报告框架不丢失。

不包含：
- 不包含保证 AI 结论完全准确。
- 不包含替代人工业务判断。
- 不包含跨问卷长期趋势分析；跨问卷分析可作为后续独立能力。
- 不包含外部 BI 系统集成。

业务规则：
- 没有答卷时不能生成真实结论，只能生成报告框架或数据需求说明。
- 报告必须明确样本量、数据范围和方法边界。
- Agent 对报告的改写必须由用户确认后才覆盖正式报告。
- 生成报告所用模型必须可见，便于追溯和复现。
- AI Ready 验收线索：读取本 UC 后，应能推导 Report Agent 面板、报告框架预览、报告章节编辑、Generate/Regenerate、对话追问、报告改写确认、导出、发送到团队、模型失败、样本不足、权限不足和数据范围变化状态。
