# 五步工作流持久壳层

## 问题

设计问卷、报告模板、发布回收、查看答题和分析报告虽然都在同一条五步工作流中，
但各步骤分别渲染了自己的页面头部、导航和宽度容器。用户点击步骤 Tab 后会看到
整页布局替换，而不是只切换 Tab 下方的工作内容。

## 用户可见行为

- 进入已有问卷后，返回列表、`Survey Workflow`、当前问卷名称和五步 Tab 只渲染一次。
- 点击任一步骤时，公共头部和五步 Tab 保持相同位置与 DOM 节点，只替换 Tab 下方内容。
- 设计问卷继续显示统一设计器；报告模板、发布回收、查看答题和分析报告继续显示各自现有业务内容。
- 当前步骤只通过选中态表达，不在公共头部重复显示独立页面标题或说明。
- 深链接刷新仍恢复对应步骤，且使用同一公共壳层。

## 非目标

- 不修改问卷、报告模板、发布规则、答卷或报告生成的数据契约。
- 不重做各步骤内部工作台。
- 不改变问卷模板库和新建问卷模板编辑器。

## 验收

- Playwright 从设计问卷依次点击报告模板、发布回收、查看答题和分析报告。
- 每次切换后 URL 与步骤内容正确，公共头部和五步 Tab 的 DOM 节点身份不变。
- 公共头部和五步 Tab 的位置、宽度在切换前后保持稳定。
- 直接刷新任一步骤深链接仍显示公共壳层和正确选中态。
