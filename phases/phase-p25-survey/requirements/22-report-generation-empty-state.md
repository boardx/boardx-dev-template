# 无答卷报告生成边界

## 背景

分析报告严格依赖问卷和真实答卷。当前问卷没有有效答卷时，点击“重新生成”会向
`professional-report` 发起请求，后端以通用 500 和
`professional_report_generation_failed` 返回，页面直接暴露内部错误码。

## 用户可见行为

- 当前事实库为 0 份有效答卷时，分析报告继续展示已有空状态或历史版本，但“重新生成”不可用。
- 页面明确提示“收到至少 1 份有效答卷后可生成报告”，并引导用户先发布和回收答卷。
- 无答卷生成请求即使绕过前端，也返回明确的业务错误，不调用模型、不发布空报告版本。
- 已有至少 1 份有效答卷时，报告生成行为保持不变，继续按已保存报告模板生成各章节。

## 非目标

- 不使用模拟答卷填充报告。
- 不修改模板章节结构、模型选择或报告导出。
- 不自动向真实问卷写入测试答卷。
