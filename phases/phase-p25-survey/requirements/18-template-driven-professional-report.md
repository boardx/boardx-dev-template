# 18 - 模板驱动的专业分析报告

## 事实来源

- 用户于 2026-07-19 对 `/surveys?survey=<id>&step=report` 实际页面提出变更：
  当前分析报告信息层级和内容表达不够专业，且最终报告没有清晰体现报告模板中的章节设计。
- 视觉参考：
  `ui-preview/2026-07-19-professional-report-reference.png`。
- 用户确认最终报告不包含预设业务章节。报告结构必须完全来自已保存的报告模板，
  每个模板章节独立生成唯一类型的产物，随后按模板顺序组装成一份完整报告。
- Umbrella issue：`boardx/boardx-dev-template#648`。本 feature 使用独立 Delivery PR，
  PR 中以 `Refs #648` 关联总需求。

## 当前问题

- 当前模型只生成一组全局 claims，服务端再用固定页面结构拼装，章节要求没有成为逐章生成任务。
- 分析报告固定展示标题、三项指标、执行摘要、方法与限制等区块，即使模板没有设计这些章节。
- 图片章节只展示生成约束，图表章节和文本章节的内容结构也没有严格受模板契约控制。
- 页面更像生成任务工作台，而不是可供管理层连续阅读、分享和导出的专业报告。
- 章节生成失败或类型不匹配时缺少原子发布边界，容易形成不完整或偏离模板的报告。

## 已确认用户体验

### 1. 模板是唯一报告结构来源

- 最终报告的章节数量、顺序、标题和输出类型完全等于生成时固化的报告模板快照。
- 不自动增加“执行摘要”“样本质量”“假设验证”“方法论”等固定业务章节。
- 用户需要上述内容时，必须在报告模板中显式创建对应章节并选择输出类型。
- 报告页可以展示标题、数据截止时间、样本量、版本和生成状态等文档元信息；
  元信息不属于业务章节，也不能改变模板结构。

### 2. 每章唯一输出

- 每个章节必须且只能生成 `text`、`chart` 或 `image` 之一。
- 文本章节输出专业正文，至少包含模板约束要求的结论、证据、边界或建议，不得回显模板提示词。
- 图表章节输出真实聚合数据、所选白名单 ECharts Option 和图表解读；
  不得使用样例数据或退化为统一静态条形图。
- 图片章节输出正式图片资产及无障碍说明；图片服务不可用时章节失败，
  不得把提示词文本冒充图片产物。
- 所有章节都从同一个只读事实库快照中自主检索整份问卷和全部授权答卷，
  不要求用户逐题绑定来源。

### 3. 章节产物组装

- 生成任务先固化模板快照和事实库修订，再按模板顺序创建章节任务。
- 每章产物必须记录 `chapterId`、`order`、`outputType`、模板要求摘要、证据引用和生成状态。
- 服务端校验章节数量、顺序、类型、图表模板和证据引用。
- 只有全部章节通过校验后，才原子发布一个不可变的完整报告版本。
- 任一章节失败时保留最近成功版本，并显示失败章节和可重试状态；不发布部分报告。
- 相同 `sourceRevision + requirementHash + templateVersion` 继续复用已有成功版本。

### 4. 专业阅读界面

- 页面参考已确认的洞察报告视觉：克制的文档标题区、清晰元信息、连续章节、充足留白和一致的图表容器。
- 左侧目录严格列出当前报告版本的模板章节，并高亮当前阅读章节。
- 中间为完整报告阅读区，按模板顺序渲染文本、图表或图片章节。
- 顶部提供返回、版本状态、重新生成、分享和导出操作；历史版本使用紧凑入口，
  不占据报告正文上方的大面积空间。
- 右侧常驻 AI 编辑栏从正式阅读页移除。修改章节要求应返回报告模板，
  避免不可变报告版本被就地修改。
- 桌面端适合连续阅读，移动端单列且无横向溢出。

## 数据契约

### Template snapshot

生成任务必须固化归一化后的模板快照：

```json
{
  "templateVersion": "survey-report-template-vNext",
  "chapters": [
    {
      "id": "chapter-id",
      "order": 1,
      "title": "章节标题",
      "outputType": "text | chart | image",
      "requirement": "自然语言要求",
      "chartTemplateId": "仅 chart 必填"
    }
  ]
}
```

### Chapter artifact

章节产物采用可判别联合，不允许一个章节同时携带多种内容：

```text
text  -> body + claims + evidenceRefs + limitations
chart -> option + interpretation + evidenceRefs + sampleSize
image -> assetUrl + altText + caption + evidenceRefs
```

- `chart.option` 只能由服务端白名单模板和真实聚合数据构造。
- `image.assetUrl` 必须指向受控持久化资产，不接受外部任意 URL。
- 原始完整答卷不进入浏览器响应；开放题证据继续执行匿名化和原文脱敏。

## 错误与版本语义

- 零样本：保留模板目录和章节空状态，不生成虚构结论或图表。
- 低样本：章节可生成方向性内容，但必须在该章节显示样本限制。
- 模型、图表或图片生成失败：整次新版本失败，保留最近成功版本。
- 模板在生成后变化：现有报告继续可读并标记“模板有更新”，用户主动重新生成新版本。
- 历史报告始终使用生成时固化的模板快照，不能被当前模板重排或改写。

## F19 范围

- 重构专业报告文档契约为模板快照和单类型章节产物。
- 让生成服务逐章消费标题、唯一输出类型、图表模板和自然语言要求。
- 增加完整章节校验和原子发布门禁。
- 按参考稿重构分析报告阅读界面和版本入口。
- 增加单元测试、API 测试、桌面/移动端 Playwright 和视觉对比证据。

## 非范围

- 不在报告阅读页编辑报告模板。
- 不新增自由布局画布或多输出章节。
- 不改变公开答题协议、问卷生命周期或权限模型。
- 不声称复刻任何咨询公司的未公开方法或内部系统。

## 验收

- 模板有 N 个章节时，成功报告恰好有 N 个章节，标题、顺序和输出类型逐项一致。
- 模板未包含执行摘要或方法论时，最终报告不得自动增加这些业务章节。
- 每章只渲染其唯一输出类型；图表使用所选 ECharts 模板和真实数据。
- 每章生成均消费该章自然语言要求，并记录来自同一事实库修订的可验证证据。
- 任一章节类型、数量、顺序或证据校验失败时，不发布不完整新版本。
- 报告页左侧目录与正文同步，历史版本入口紧凑，右侧无常驻编辑助手。
- 零样本、低样本、生成失败和模板过期状态可见且不伪造结论。
- 浏览器响应不包含完整原始答卷。
- 桌面和移动端无文字重叠或横向溢出。
