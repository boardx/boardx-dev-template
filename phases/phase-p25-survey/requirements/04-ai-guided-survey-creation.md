Use Case 名称：
使用智能 Agent 引导创建问卷

Actor：
Survey Owner、Survey Admin、Survey Member

目标：
用户用自然语言说明调研目标后，智能 Survey Agent 通过追问、建议和结构化草稿，帮助用户生成一份可编辑、可预览、可发布的针对性问卷。

系统边界：
Survey / Survey Agent / Survey Builder

前端入口：
- Survey 列表 > Create with AI。
- Survey Builder > AI Assistant 面板。
- Survey 工作区中选择“创建问卷”能力。

前置条件：
- 用户已登录。
- 当前团队上下文已加载。
- 用户拥有创建问卷的权限。
- 至少有一个可用 LLM 模型配置；如果没有可用模型，AI Assistant 显示配置缺失状态。
- 题型库、模板库和 Survey Builder 可以被 Agent 调用或引用。

触发条件：
用户输入自然语言目标，例如“我要创建一份商品反馈问卷”，并点击 Generate / Start with AI。

主流程：
1. 用户点击 Create with AI。
2. 系统打开智能 Survey Agent 面板，展示输入框、模型选择器、示例提示词、当前团队上下文和“开始生成”按钮；如果用户已在 Builder 中，AI 面板以侧栏形式打开，不替代 Questions 画布。
3. 用户输入调研目标，例如“我要创建一份商品反馈问卷，用来了解用户对新品包装和价格的反馈”。
4. Agent 解析目标，展示理解摘要，包括调研对象、调研目的、预期输出、推荐问卷类型和缺失信息。
5. Agent 针对缺失信息提出 3 到 7 个澄清问题，例如目标受众、商品类别、反馈维度、是否需要 NPS、是否匿名、预计答题时间、是否收集联系方式。
6. 用户逐项回答澄清问题，也可以跳过问题、修改 Agent 对目标的理解，或要求“直接生成第一版”。
7. Agent 根据用户回答生成问卷方案文档，包含问卷标题、说明、建议使用场景、目标人群、预计答题时长、题目分类、题目列表、题型、必填规则、逻辑跳转建议和发布策略建议。
8. 系统把生成的题目按分类展示为可编辑结构，例如“购买背景”“商品体验”“价格与包装”“复购意愿”“开放反馈”。
9. 用户可以直接编辑题目标题、说明、选项、题型、必填状态、分类名称和分类顺序。
10. 用户可以拖拽改变题目顺序，也可以把题目移动到其他分类。
11. 用户可以要求 Agent 执行局部修改，例如“问题太多，控制在 8 题以内”“加一个包装满意度评分”“把开放题放到最后”“改成 B2B 客户语气”。
12. Agent 生成修改建议时，以 diff、候选卡片或“待应用变更”形式展示，用户确认后才应用到问卷草稿。
13. 用户可以要求 Agent 检查当前问卷是否符合 Google Forms 常见发布标准，例如是否有标题、题目数量是否合理、是否配置必填、是否存在无效跳转、是否需要收集邮箱。
14. 用户点击 Preview Survey，系统以答题人视角展示生成问卷；预览中展示题目、分页、进度条、必填校验、逻辑跳转和提交成功页。
15. 用户点击 Preview Report Outline，系统基于当前问卷结构展示未来报告大纲，包括样本概览、各分类分析、关键指标、开放反馈摘要、建议行动和风险提示。
16. 用户确认草稿后点击 Apply to Builder，系统把问卷方案转入 Survey Builder，形成可继续手工编辑的真实问卷草稿。
17. 用户点击发布入口时，Agent 可以给出发布建议，但正式发布仍由用户在发布面板中确认。
18. 用户点击 Create survey 保存；保存成功后问卷出现在对应列表。

备选流程：
- A1：用户只输入一句很模糊的目标，Agent 先生成澄清问题，不直接生成长问卷。
- A2：用户选择“快速模式”，Agent 生成短问卷版本，默认控制题目数量和答题时长。
- A3：用户选择“专业模式”，Agent 增加调研假设、维度设计、题目意图、报告指标和逻辑跳转建议。
- A4：用户从已有模板开始，Agent 先解释模板适用性，再按用户目标改写模板。
- A5：用户上传或粘贴商品说明、活动文案、用户访谈摘要，Agent 提取关键维度并生成题目。
- A6：用户不满意生成结果，点击 Regenerate，系统保留原版本，并生成新版本供对比。
- A7：用户把 AI 生成的某个分类保存为模板片段，后续可复用。

异常流程：
- E1：模型不可用或调用失败，Agent 面板展示模型失败状态、重试入口和切换模型入口；已输入内容和已生成草稿不丢失。
- E2：Agent 生成内容违反问卷结构约束，例如没有题型、选项为空、逻辑跳转指向不存在题目，系统标记无效项并禁止 Apply to Builder。
- E3：用户没有创建权限时，Create with AI 入口隐藏；若直接打开 AI 创建页，页面显示无权限状态。
- E4：用户请求超出系统边界，例如“自动帮我发送给所有用户并抽奖”，Agent 解释当前不支持，并把可支持部分转成问卷草稿。
- E5：生成内容过长时，系统提示拆分或压缩，并提供“控制题量”“拆成多份问卷”的建议。

权限与可见性：
- Survey Owner、Survey Admin、允许创建的 Survey Member 可以使用 AI 创建问卷。
- Respondent 不能使用创建问卷 Agent。
- Visitor/未加入团队的用户不能进入 AI 创建流程。
- 团队上下文中的敏感信息只用于当前用户有权限访问的问卷创建，不应暴露给无权限用户。

后置条件：
- 成功时，用户获得一份结构化问卷草稿，包含可编辑题目、分类、顺序、题型、选项、逻辑和发布建议。
- Apply to Builder 后，AI 草稿成为真实 Survey Builder 中的可编辑问卷结构。
- 保存成功后，问卷出现在 My Surveys 或 Team Surveys 列表。
- 失败时，用户输入、澄清回答和已生成草稿保留，便于重试或切换模型。

不包含：
- 不包含自动代替用户发布问卷。
- 不包含自动向外部渠道投放问卷。
- 不包含保证 AI 生成的问题完全科学或无偏见。
- 不包含外部商品数据抓取，除非用户主动提供材料。

业务规则：
- Agent 生成的内容必须转成可编辑结构，不能只返回一段不可编辑文本。
- Agent 的修改建议必须由用户确认后才应用到问卷草稿。
- Agent 必须服务于主编辑器：Questions 画布、Responses 页和 Settings 页仍是正式工作区；AI 面板只能生成建议、草稿和检查结果，不能绕过用户确认直接发布或覆盖问卷。
- Agent 的待应用变更必须能映射到具体结构动作：新增题目、删除题目、改题型、改选项、移动分类、移动题目、配置必填、配置跳转、补充发布策略或补充报告大纲。
- Agent 应能解释每个建议对应的目标，例如“减少答题疲劳”“便于价格敏感度分析”“符合发布前检查”“便于 Summary 汇总”。
- 所有题目必须落到系统支持的题型库；无法映射的题目必须标记为需人工处理。
- 报告大纲预览基于当前问卷结构生成，不需要真实答卷。
- AI Ready 验收线索：读取本 UC 后，应能推导 Create with AI 入口、Google Forms 风格 Builder 旁路 AI 面板、模型选择器、澄清问题、问卷方案文档、待应用变更、分类题目编辑、拖拽排序、局部改写、发布前检查建议、问卷预览、报告大纲预览、Apply to Builder、模型失败和无权限状态。
