Use Case 名称：
切换 Survey AI 使用的 LLM 模型

Actor：
Survey Owner、Survey Admin

目标：
用户或管理员为 Survey Agent 和 Report Agent 选择可用 LLM 模型，并在生成失败、质量不满意或成本/速度需求变化时切换模型。

系统边界：
Survey / AI Model Gateway / Survey Agent

前端入口：
- Survey Agent 面板顶部的 Model selector。
- Report Agent 面板顶部的 Model selector。
- Survey AI Settings > Survey AI Models。

前置条件：
- 用户已登录。
- 至少配置了一个模型供应商，或用户有权限进入模型配置页面。
- 系统支持的模型供应商包括 OpenAI、Gemini、通义千问、MiniMax，以及后续可扩展供应商。

触发条件：
用户打开模型选择器、切换模型、保存默认模型、模型调用失败后点击 Switch model，或管理员配置模型供应商。

主流程：
1. 用户打开 Survey Agent 或 Report Agent。
2. 面板顶部展示当前模型名称、供应商、速度/质量/成本提示和模型选择器。
3. 用户点击模型选择器，系统展示可用模型列表，按供应商分组：OpenAI、Gemini、通义千问、MiniMax、自定义模型。
4. 每个模型显示能力标签，例如适合问卷生成、适合报告分析、长上下文、多语言、低成本、快速响应、支持结构化输出。
5. 用户选择一个模型后，系统把后续 Agent 请求发送到该模型，并在生成结果中展示使用的模型名称。
6. 用户可以点击 Set as default，把该模型设为当前团队的 Survey AI 默认模型；无管理员权限时只保存为个人偏好。
7. 当模型调用失败时，系统展示失败状态、Retry、Switch model 和使用备用模型的选项。
8. 用户切换模型后重新生成，系统保留原提示词、澄清回答、问卷草稿、报告框架或报告上下文，不要求用户重新输入。
9. 管理员进入 Survey AI Settings，配置供应商启用状态、API Key、默认模型、可用模型、成本限制、数据出境提示和备用模型顺序。
10. 保存配置后，Survey Agent 和 Report Agent 的模型列表同步更新。

备选流程：
- A1：用户选择自动模式，系统按任务自动选择模型：问卷生成优先结构化输出，报告分析优先长上下文和推理质量。
- A2：用户选择低成本模式，系统优先使用成本较低模型，并提示可能影响质量。
- A3：用户选择高质量模式，系统优先使用更强模型，并提示成本或速度影响。
- A4：某供应商未配置 API Key，该供应商模型展示为不可用，并提供配置入口给管理员。
- A5：用户要求对比生成，系统用两个模型生成候选结果，并让用户选择保留版本。

异常流程：
- E1：所有模型不可用，Agent 面板显示“暂无可用模型”，并提供联系管理员或进入配置入口。
- E2：模型超时，系统允许重试、切换模型或保留当前草稿。
- E3：模型返回非结构化内容，系统尝试结构化修复；修复失败时标记需人工处理。
- E4：用户没有模型配置权限时，AI Settings 入口隐藏或只读。
- E5：供应商配置保存失败，页面保留用户输入并显示失败原因。

权限与可见性：
- Survey Admin 可以配置供应商、API Key、默认模型、备用模型和成本策略。
- Survey Owner/Admin 可以在允许范围内选择本次生成使用的模型。
- 普通 Survey Member 可以使用默认模型；是否允许手动切换由团队策略决定。
- Respondent 不看到模型选择器。
- Visitor/未加入团队的用户不能访问模型配置或 Agent。

后置条件：
- 模型切换成功后，后续 Survey Agent 或 Report Agent 调用使用新模型。
- 默认模型保存成功后，新会话优先使用该模型。
- 生成结果记录模型供应商、模型名称和生成时间。
- 模型失败时，用户草稿和上下文不丢失。

不包含：
- 不包含具体供应商 SDK 实现。
- 不包含模型训练或微调。
- 不包含供应商计费结算系统。
- 不包含绕过团队数据安全策略的模型调用。

业务规则：
- 每次 AI 生成都必须记录使用的模型，便于追踪质量问题。
- 模型选择不能改变用户权限；无结果权限的用户不能通过切换模型读取报告数据。
- 备用模型只在用户或团队策略允许时启用。
- 对结构化问卷生成，模型输出必须经过结构化校验后才能应用到 Survey Builder。
- AI Ready 验收线索：读取本 UC 后，应能推导模型选择器、供应商分组、模型能力标签、默认模型、自动/低成本/高质量模式、失败切换、管理员配置、权限差异、结果追踪和结构化输出校验。
