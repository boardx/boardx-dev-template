# Commercial AI Survey System

## 原始需求

构建一整套商业版问卷系统。问卷系统要集成 AI 智能体，尽可能通过智能体完成整个问卷系统的操作，包括设计问卷、优化问卷、生成和输出报告。智能体需要理解操作人提供的信息，完成任务；过程中遇到不确定信息时可以多轮反馈，以便得到精准、高效的问卷与报告。

## 产品目标

- 让用户从“我想做一次调研”开始，通过 AI Agent 完成问卷目标澄清、问卷设计、题目生成、优化建议、发布前检查和报告生成。
- 保留商业产品所需的可控性：AI 只生成建议、草稿和待应用变更，关键动作必须由用户确认。
- 保留完整追溯：每次 AI 操作都记录输入、模型、输出、待应用变更、用户确认结果和证据。
- 让 PM、开发、QA、评审 agent 在合适阶段进入，而不是所有 agent 同时写代码。

## 核心用户路径

1. 用户进入 Survey，选择 `Create with AI`。
2. Survey Agent 询问调研目标、目标人群、业务背景、题量、匿名性、报告用途等关键问题。
3. 用户可以逐轮回答，也可以要求“先生成一版”。
4. Agent 输出结构化问卷方案，包括标题、说明、维度、题目、题型、选项、必填规则、跳转建议、发布建议和报告大纲。
5. 用户要求优化，例如“更短”“更专业”“加入 NPS”“面向 B2B 客户”“适合活动报名后回访”。
6. Agent 以待应用变更形式展示修改，用户确认后应用到 Builder。
7. 用户预览答题页和报告大纲，保存或发布问卷。
8. 回收答卷后，用户在 Report 中让 Report Agent 生成报告、追问数据含义、改写报告口径，并在问卷系统内导出报告。

## 商业版能力分层

### L1 基础问卷系统

已由 F01-F06 覆盖：创建、管理、答题、报告、模板、发布暂停。

### L2 AI 设计与优化

- 多轮澄清：Agent 不确定时先问，而不是乱生成。
- 结构化草稿：AI 输出必须能落入真实题型库。
- 待应用变更：新增题目、删除题目、改题型、改选项、移动分类、配置必填或跳转，都必须让用户确认。
- 发布前检查：检查标题、题量、题型覆盖、必填、跳转、答题时长、报告指标。

### L3 AI 报告与输出

- 无答卷时只能生成报告框架，不能编造结论。
- 有答卷时必须展示样本量、数据范围、生成时间、模型名称、证据引用和不确定性说明。
- 支持报告改写、压缩、展开、语气转换，并在问卷系统内导出 PDF/CSV。

### L4 模型与治理

- Survey Agent / Report Agent 支持模型选择、默认模型、自动模式、失败切换和备用模型。
- Survey Admin 可以管理供应商、API Key、默认模型、成本策略、数据出境提示。
- 所有 AI 结果记录模型供应商、模型名称和生成时间。

## 非目标

- AI 不自动替用户发布问卷。
- AI 不自动向外部渠道投放问卷。
- AI 不保证调研方法完全科学；必须暴露方法边界。
- AI 不绕过权限读取问卷或答卷数据。
- 不包含任何非问卷系统入口、分发或协作模块；本阶段只做独立 Survey 系统。
- Respondent 不访问创建、优化或报告 Agent。

## 验收原则

- 每个 AI feature 都必须有可执行 verification 命令。
- 每个 AI 操作必须有失败态、重试、模型切换或降级路径。
- AI 输出必须结构化校验后才能应用到 Survey Builder。
- 用户确认前，Agent 不能覆盖正式问卷或正式报告。
- 没有 evidence 的实现不算完成。
