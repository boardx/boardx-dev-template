=== Root cause analysis ===
Confirmed via direct psql query + manual page.evaluate(fetch(...)) that:
1. The listKbFiles SQL (count + paginated SELECT) executes instantly (<10ms) against Postgres directly.
2. A manual fetch('/api/kb/files?...') issued via page.evaluate() resolves correctly with expected JSON.
3. The SAME fetch call, when triggered from the page's useEffect/load(), intermittently never resolves
   within the 10s default Playwright expect-timeout -- request stays pending (REQ START logged, no
   REQ FINISHED/REQ FAILED ever logged) purely due to Next.js dev-server / Node event loop being
   starved by host-level CPU contention (load average observed climbing from ~19-28 to 56-76 on an
   8-core machine over the course of ~15 minutes, with 35+ concurrent node processes / many parallel
   agent worktrees each running their own next dev + docker compose stacks).
4. Failure count is non-deterministic and worsens as host load increases across repeated runs
   (2 failures -> 3 failures on immediate re-run), which is inconsistent with a deterministic code bug
   and consistent with resource starvation.
