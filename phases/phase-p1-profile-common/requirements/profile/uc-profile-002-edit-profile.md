Use Case 名称：
编辑个人信息和头像

Actor：
注册用户

目标：
用户在账号中心维护显示名和头像，并能从候选头像或 AI 生成头像中选择一个作为个人头像。

系统边界：
BoardX / Account Center / Personal info

前端入口：
1. 用户菜单中的 Profile / 个人资料入口。
2. 账号中心的 Personal info 分区。

前置条件：
用户已登录并进入账号中心的 Personal info 分区。

触发条件：
用户修改显示名、选择头像或点击 AI generate 后保存个人信息。

主流程：
1. 用户从 Profile / 个人资料入口进入账号中心。
2. 系统打开 Personal info 分区，并展示当前邮箱、显示名输入框、当前头像预览、候选头像列表、AI generate 入口和 Save personal info 按钮。
3. 当前头像预览显示用户已保存头像；如果用户没有头像，系统展示默认生成头像或姓名首字母。
4. 候选头像列表展示多个可点击的头像选项；每个选项应能被选中，并用选中态标识当前选择。
5. 用户点击候选头像，系统立即更新当前头像预览，但尚不保存到账号资料。
6. 用户点击 AI generate，系统基于当前显示名、邮箱或用户标识生成一个新的 AI 风格头像，并立即更新当前头像预览。
7. 用户修改 Display name 输入框，系统保留用户输入。
8. 用户点击 Save personal info。
9. 系统校验 Display name 不能为空；如果为空，系统在当前区域展示错误，且不提交保存。
10. 校验通过后，保存按钮进入提交状态，系统保存显示名和头像。
11. 保存成功后，系统更新当前会话中的用户资料、刷新用户菜单中的头像和显示名，并在 Personal info 区域展示成功提示。
12. 保存失败时，系统展示错误信息，保留用户当前输入和头像选择，用户可修改后重试。

备选流程：
- A1：用户只修改显示名，不修改头像。
- A2：用户只选择头像，不修改显示名。
- A3：用户连续点击多个候选头像，系统始终只保留最后一次选择作为待保存头像。
- A4：用户点击 AI generate 多次，系统每次生成新的头像预览，最后一次结果作为待保存头像。
- A5：用户离开账号中心但未保存，系统不更新账号资料。

异常流程：
- E1：Display name 为空，系统提示显示名必填。
- E2：头像生成失败，系统提示生成失败，并保留原头像或上一次可用预览。
- E3：头像资源加载失败，系统展示默认头像占位，用户仍可选择其它头像。
- E4：保存失败，系统提示重试，且不覆盖已保存的个人资料。
- E5：登录状态失效，系统跳转到登录页。

权限与可见性：
1. 注册用户只能管理自己的个人资料。
2. 访客不能进入账号中心。
3. 用户不能编辑其它用户的头像、显示名或邮箱。

后置条件：
成功时，用户资料中的显示名和头像被更新；失败时，原资料保持不变。

不包含：
1. 不包含 Team 成员资料管理。
2. 不包含系统管理员编辑用户。
3. 不包含邮箱修改流程。
4. 不包含密码修改流程。

业务规则：
1. 显示名不能为空。
2. 头像必须来自系统允许的候选头像、用户已授权上传资源或系统生成资源。
3. AI 生成头像只负责生成可选头像，不自动保存为用户头像。
4. 保存成功后，当前会话中展示的头像和显示名必须与账号资料保持一致。
