Use Case 名称：
管理团队 Memory

Actor：
团队 owner、团队 admin

目标：
团队管理者进入团队 Memory 页面，维护团队在 AI 协作中可复用的上下文信息。

系统边界：
BoardX / Team Memory

前端入口：
Team 管理页左侧菜单中的 Memory。

前置条件：
- 用户已登录并已选择当前团队。
- 用户可以进入 Team 管理页。

触发条件：
用户点击 Memory 菜单项。

主流程：
1. 用户在 Team 管理页左侧点击 Memory。
2. 系统进入当前团队的 Memory 页面，并保持 Team 管理页导航可见。
3. 系统展示团队 Memory 列表、搜索输入、新增 Memory 输入区和新增按钮；加载时先展示占位状态。
4. 用户可以输入关键字搜索，系统只显示匹配的 Memory，并保留总数和过滤结果状态。
5. 用户在新增输入区输入内容，可以按 Enter 或点击新增按钮保存；Shift+Enter 用于在输入中换行。
6. 如果内容为空，系统不新增；如果内容已存在，系统提示已存在。
7. 新增成功后，系统把 Memory 加入列表并显示成功提示。
8. 用户点击某条 Memory 的删除入口，系统打开确认弹窗。
9. 用户确认删除后，系统从列表移除该 Memory 并显示删除成功提示。
10. 保存失败时，系统回退本次新增或删除，并提示保存失败。

备选流程：
- A1：用户只查看 Memory 内容，不做修改。
- A2：用户从 Memory 切换到其他团队管理页面。

异常流程：
- E1：如果团队信息缺失，页面无法准确关联当前团队。

权限与可见性：
- owner/admin 可以通过 Team 管理页进入 Memory。
- member 不能通过 Manage Team 入口进入该页面；本用例不授予 member 管理团队 Memory 的能力。
- visitor/未加入团队的用户不能进入团队 Memory 页面。

后置条件：
- 保存成功时，团队 Memory 更新。
- 未保存时，团队 Memory 保持原状态。

不包含：
- 不包含 Memory 的具体字段、排序和冲突处理。
- 不包含 AI 模型如何使用 Memory 的内部过程。

业务规则：
- Memory 内容按文本去重，保存前后列表按文本排序展示。
