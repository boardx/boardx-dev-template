Use Case 名称：
创建团队

Actor：
已登录用户

目标：
用户在 BoardX 中创建一个新的协作团队，后续可以在该团队下进入房间、成员管理、知识库、AI Store 和问卷等工作区。

系统边界：
BoardX / Team

前端入口：
1. 左侧团队头像菜单中的 Create Team。
2. 用户没有任何团队时自动出现的创建团队弹窗。

前置条件：
- 用户已登录。
- 用户当前没有团队，或用户主动打开团队菜单。

触发条件：
用户点击 Create Team，或系统发现用户没有任何团队并要求先创建团队。

主流程：
1. 用户打开团队头像菜单，看到当前团队、自己的团队角色、Manage Team、Team Knowledge Base、Create Team 以及团队列表。
2. 用户点击 Create Team，系统打开创建团队弹窗。
3. 弹窗中展示团队名称输入框和创建按钮；如果用户尚无任何团队，系统不允许直接关闭该弹窗。
4. 用户输入团队名称，可以按 Enter 或点击创建按钮提交。
5. 系统检查团队名称不能为空。
6. 系统创建团队，创建期间按钮显示创建中状态并避免重复提交。
7. 创建成功后，系统提示团队创建成功，把新团队保存为当前团队，并刷新团队列表。
8. 系统清理与旧团队相关的 AVA 会话状态，并保持应用处于新团队上下文。
9. 用户再次打开团队菜单时，可以看到新团队出现在团队列表中。

备选流程：
- A1：用户已有团队时，可以取消创建并回到当前团队。
- A2：用户没有任何团队时，关闭弹窗不会生效，用户需要先创建团队。

异常流程：
- E1：团队名称为空，系统提示团队名称为空，弹窗保持打开。
- E2：创建失败，系统提示创建团队失败，弹窗保留用户输入。

权限与可见性：
- 创建者成为该团队的 owner。
- owner/admin 进入团队菜单时可以使用 Manage Team。
- member 点击 Manage Team 时，系统提示其没有权限进入团队管理面板。
- 未登录用户不能创建团队；需要先完成登录。

后置条件：
- 成功时，新团队成为当前团队。
- 成功时，团队列表包含新团队。
- 失败时，不创建团队，用户仍停留在弹窗中。

不包含：
- 不包含后台系统管理员创建团队。
- 不包含团队计费、成员管理或房间创建。

业务规则：
- 团队名称不能为空。
- 创建团队成功后，创建者成为该团队的 Owner，并可进入新团队空间继续配置团队信息。
