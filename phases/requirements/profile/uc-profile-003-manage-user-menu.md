Use Case 名称：
使用用户菜单

Actor：
注册用户

目标：
用户通过个人菜单进入账号中心的个人资料或设置分区，并继续访问个人知识库、邀请好友、升级或 Credit、语言、主题、退出登录等账号相关能力。

系统边界：
BoardX / User Menu

前端入口：
1. 页面中的头像或用户菜单入口。
2. 侧边栏或顶部导航中的账号入口。

前置条件：
用户已登录。

触发条件：
用户打开页面中的用户菜单。

主流程：
1. 用户在页面右上角或侧边栏看到头像按钮。
2. 用户点击头像按钮，系统展开用户菜单。
3. 非简化菜单顶部展示当前团队名称、用户头像、姓名、邮箱或基础身份信息，以及可用的升级或 Credit 入口。
4. 系统展示账号相关菜单项：Profile / 个人资料、Settings / 设置、个人知识库，以及在非 Credit 计费模式下的升级或管理计划入口。
5. 系统展示邀请好友入口和退出登录入口。
6. 系统展示语言区域，列出 English 和中文，并标识当前语言。
7. 如果当前视图启用主题切换，系统展示浅色、深色和系统主题选项，并标识当前主题。
8. 用户点击 Profile / 个人资料，系统进入账号中心并打开 Personal info 分区。
9. 用户点击 Settings / 设置，系统进入账号中心并打开 Settings 分区。
10. 用户点击个人知识库、邀请好友、升级/Credit 或记录入口时，系统打开对应页面或弹窗。
11. 用户点击退出登录，系统清除登录状态并跳转到当前语言对应的 signin 页面。

备选流程：
- A1：用户关闭菜单，不执行操作。
- A2：移动端以抽屉或菜单形式展示。

异常流程：
- E1：登录状态失效，系统要求重新登录。
- E2：某入口不可用或未启用时，系统不展示该入口或不允许继续操作。

权限与可见性：
1. 只有注册用户可以看到完整用户菜单。
2. 访客只能看到登录或注册入口。

后置条件：
用户进入所选账号能力，或继续停留在当前页面。

不包含：
1. 不包含各账号能力内部流程。
2. 不包含认证会话内部流程。

业务规则：
1. 用户菜单展示的信息必须属于当前登录用户。
2. 退出登录必须清除当前会话并返回安全页面。
3. Profile 和 Settings 入口必须进入账号中心的不同默认分区，不能表现为完全相同的跳转结果。
