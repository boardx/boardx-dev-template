Use Case 名称：
查看账号中心

Actor：
注册用户

目标：
用户从用户菜单进入账号中心，查看自己的个人信息、安全设置和偏好设置入口。

系统边界：
BoardX / Account Center

前端入口：
1. 用户菜单中的 Profile / 个人资料入口。
2. 用户菜单中的 Settings / 设置入口。
3. 需要管理账号资料或安全信息的受保护页面跳转。

前置条件：
用户已登录 BoardX。

触发条件：
用户点击用户菜单中的 Profile、Settings，或访问账号中心地址。

主流程：
1. 用户打开用户菜单。
2. 系统在用户菜单中展示当前用户头像、显示名或邮箱，并展示 Profile、Settings、退出登录等账号入口。
3. 用户点击 Profile / 个人资料。
4. 系统进入账号中心并默认打开 Personal info 区域。
5. 系统展示账号中心标题、返回工作区入口、左侧分区导航，以及 Personal info、Security、Settings 三个可切换入口。
6. Personal info 区域展示当前邮箱、显示名输入框、当前头像预览、可选头像列表、AI generate 头像入口和保存按钮。
7. 用户点击 Settings / 设置入口时，系统进入同一个账号中心，但默认打开 Settings 区域。
8. Settings 区域展示账号偏好设置项，例如 AI 模型偏好、默认隐私级别和保存按钮。
9. 用户点击 Security 区域时，系统展示修改密码表单，包含当前密码、新密码、确认新密码和提交按钮。
10. 用户点击 Back to workspace / 返回工作区，系统返回用户进入账号中心前的工作区或默认工作区。
11. 用户切换分区时，系统只切换账号中心内容区，不退出登录，也不丢失当前已加载的用户资料。

备选流程：
- A1：用户直接访问账号中心但未带分区参数，系统默认打开 Personal info。
- A2：用户通过 Settings 入口进入，系统直接打开 Settings 分区。
- A3：用户在移动端进入账号中心，分区导航可以折叠为顶部列表或下拉入口，但必须能切换 Personal info、Security、Settings。
- A4：资料不完整时，系统在对应区域展示可补充字段。

异常流程：
- E1：用户未登录，系统跳转到登录页，并保留登录后的返回目标。
- E2：当前用户资料加载失败，系统提示刷新或重新登录。
- E3：某个设置能力未启用时，系统不展示不可操作入口，或展示明确的不可用状态。

权限与可见性：
1. 注册用户只能查看和管理自己的账号中心。
2. 访客不能进入账号中心。
3. Profile 和 Settings 是进入同一账号中心的不同分区入口，不应跳过分区切换交互。

后置条件：
用户看到账号中心，并可继续编辑个人信息、修改密码或保存偏好设置。

不包含：
1. 不包含 Team 成员资料管理。
2. 不包含系统管理员编辑用户。
3. 不包含第三方身份提供商的账号管理页面。

业务规则：
1. 个人资料仅登录用户可访问。
2. 敏感账号信息应遵守隐私策略。
3. 用户菜单中的 Profile 和 Settings 必须能让用户明确知道自己进入的是账号中心的不同功能分区。
