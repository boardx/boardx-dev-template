Use Case 名称：
管理个人 Memory

Actor：
注册用户

目标：
用户维护个人 Memory，使 AI 在个人上下文中使用用户偏好、事实和长期信息。

系统边界：
BoardX / Profile / User Memory

前端入口：
1. 用户菜单中的个人 Memory 入口。
2. 个人设置中的 Memory 入口。

前置条件：
用户已登录。

触发条件：
用户打开个人 Memory 入口并新增、编辑或删除 Memory。

主流程：
1. 用户从个人 Memory 入口或相关设置入口打开个人 Memory 页面或面板。
2. 系统加载当前用户的 Memory 列表，并展示搜索框、输入区、添加按钮、空状态或现有 Memory 项。
3. 用户在输入区输入个人偏好、事实或长期信息。
4. 用户点击添加按钮，或在输入区按 Enter 提交；Shift+Enter 用于换行。
5. 系统校验内容不能为空，并检查是否与已有 Memory 重复。
6. 保存中系统禁用重复保存；保存成功后列表按文本排序展示新 Memory，并显示成功反馈。
7. 用户输入搜索关键词时，系统按关键词过滤当前 Memory 列表。
8. 用户点击删除入口，系统打开确认弹窗。
9. 用户确认删除后，系统保存更新后的列表，成功后移除该 Memory 并展示成功反馈。
10. 保存失败时，系统回退本地列表并展示错误反馈。

备选流程：
- A1：用户删除不再需要的 Memory。
- A2：用户只查看 Memory，不做修改。

异常流程：
- E1：Memory 内容为空，系统阻止保存。
- E2：保存失败，系统保留输入并提示重试。

权限与可见性：
1. 用户只能管理自己的个人 Memory。
2. 其他用户和 Team 成员不能查看个人 Memory，除非用户主动授权。

后置条件：
个人 Memory 被创建、更新或删除。

不包含：
1. 不包含 Team Memory 和 Board Memory。
2. 不包含 AI 检索规则。

业务规则：
1. 个人 Memory 默认属于当前用户私有数据。
2. 删除后后续 AI 上下文不应继续引用该 Memory。
