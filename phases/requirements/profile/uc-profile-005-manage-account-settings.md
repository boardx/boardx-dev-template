Use Case 名称：
管理账号设置

Actor：
注册用户

目标：
用户在账号中心维护自己的默认偏好，使后续使用 AVA、知识库、Room、Board 或其它工作区能力时采用合适的个人默认设置。

系统边界：
BoardX / Account Center / Settings

前端入口：
1. 用户菜单中的 Settings / 设置入口。
2. 账号中心左侧 Settings 分区入口。

前置条件：
用户已登录 BoardX。

触发条件：
用户点击 Settings / 设置入口，或在账号中心切换到 Settings 分区。

主流程：
1. 用户打开用户菜单。
2. 系统展示 Settings / 设置入口。
3. 用户点击 Settings / 设置。
4. 系统进入账号中心，并默认打开 Settings 分区。
5. 系统展示账号中心标题、返回工作区入口、左侧分区导航，并高亮 Settings 分区。
6. Settings 分区展示可编辑的个人偏好设置项。
7. 系统展示 AI model preference 下拉选择，用户可以选择 Auto、Local first、BoardX cloud 等可用选项。
8. 系统展示 Default privacy 下拉选择，用户可以选择 Local only、Confidential、Internal、Cloud allowed、Public 等可用选项。
9. 用户修改一个或多个设置项后，点击 Save settings。
10. 系统校验所选值必须属于当前允许的选项集合；不合法时展示错误并阻止保存。
11. 校验通过后，保存按钮进入提交状态，系统保存用户设置。
12. 保存成功后，系统在 Settings 分区展示成功提示，并保留用户最新选择。
13. 用户切换到 Personal info 或 Security 分区时，系统展示对应分区内容，已保存设置不丢失。

备选流程：
- A1：用户进入 Settings 只查看当前设置，不做修改。
- A2：某项设置受团队或企业策略限制时，系统展示当前值和限制说明，并禁用不可编辑选项。
- A3：移动端账号中心可以把左侧分区导航折叠为顶部切换入口，但必须能明确进入 Settings 分区。
- A4：用户点击 Back to workspace，系统返回工作区，不再停留在账号中心。

异常流程：
- E1：用户未登录访问 Settings，系统跳转到登录页，并保留登录后的返回目标。
- E2：设置加载失败，系统展示加载失败提示，用户可刷新或重试。
- E3：保存失败，系统展示错误提示，用户当前选择保留在页面上，已保存设置不被覆盖。
- E4：用户选择了当前不支持的设置值，系统提示该选项不可用。

权限与可见性：
1. 注册用户只能查看和修改自己的账号设置。
2. 访客不能进入账号设置。
3. 受团队或企业策略控制的设置必须显示为受限或不可编辑状态。

后置条件：
成功时，用户账号设置被更新；失败时，原设置保持不变。

不包含：
1. 不包含 Team 设置、Room 设置或 Board 设置。
2. 不包含系统管理员替用户修改设置。
3. 不包含 AI 模型服务内部调用逻辑。

业务规则：
1. Settings 是账号中心的独立分区，不应与 Personal info 混在同一个不可区分的表单里。
2. 用户可选设置项必须来自系统允许的枚举或策略配置。
3. 保存设置只影响当前用户的个人默认偏好，不自动修改已存在 Team、Room 或 Board 的权限策略。
