Use Case 名称：
打开最近访问白板

Actor：
已登录用户

目标：
用户从首页快速回到最近编辑或查看的白板。

系统边界：
BoardX / Home

前端入口：
1. Home 页面中出现的最近白板入口。
2. Recent 页面中出现的最近白板入口。

前置条件：
- 用户已登录。
- 用户有最近访问白板记录。

触发条件：
用户在首页点击最近白板卡片。

主流程：
1. 用户进入当前 Home Page，系统展示 Agent Home、Agent 分组、推荐功能和当前可见的继续入口。
2. 系统不把最近白板列表作为当前 Home Page 必然展示内容。
3. 用户如果在旧版 Recent Boards 组件、Room 页面或其它页面看到最近白板卡片，卡片展示白板名称和日期。
4. 用户点击某个白板卡片，系统按卡片携带的白板标识跳转到 Board 页面。
5. Board 页面加载目标白板，并展示 Header、画布内容、协作状态和当前角色可用入口。
6. 如果用户只有只读权限，系统进入白板查看状态，并隐藏或禁用编辑入口。
7. 如果当前 Home Page 没有最近白板入口，用户需要通过 Room、Board、全局搜索或其它已展示入口继续访问真实白板资源。

备选流程：
- A1：最近白板为空，系统展示创建或进入房间的入口。
- A2：最近白板入口尚未在当前页面启用时，系统展示开发中、空状态或不展示该入口。

异常流程：
- E1：白板已删除，系统提示资源不存在。
- E2：用户失去访问权限，系统拒绝访问。

权限与可见性：
1. 注册用户可以查看首页工作台。
2. 未登录用户应进入身份认证入口。

后置条件：
- 成功时，用户进入目标白板。

不包含：
1. 不包含 Board 内部编辑。
2. 不包含 Team 设置管理。
3. 不包含 Admin 后台。

业务规则：
- 最近白板应按团队和 用户标识 查询。
- 已删除或无权限白板不应继续展示或应在点击时提示。
- Home 与 Recent 页面只展示当前页面实际返回的最近白板；没有最近白板时，系统显示空状态，不生成虚假列表。
