Use Case 名称：
查看 Home Page

Actor：
已登录用户

目标：
用户进入 Home Page 后，查看当前团队上下文下的 Agent 工作入口，并快速开始 AVA 相关工作。

系统边界：
BoardX / Home

前端入口：
1. 应用左侧导航中的 Home。
2. 登录后默认进入的 Home Page。
3. 浏览器直接访问当前语言下的 Home 地址。

前置条件：
- 用户已登录。
- 用户已选择或拥有默认团队。

触发条件：
用户登录后进入 Home Page，或从左侧导航返回 Home Page。

主流程：
1. 用户进入当前语言下的 Home Page。
2. 系统先展示页面加载状态；客户端初始化完成后展示 Agent Home 页面。
3. 系统加载当前用户、当前团队、我的订阅 Agent、团队订阅 Agent、最近使用 Agent 和可用 Agent 数据。
4. 系统在顶部欢迎区域展示用户名称，并在有团队名称时展示当前团队名称。
5. 系统展示搜索框，用户可按 Agent 名称、描述或标签搜索当前页面可见的 Agent。
6. 如果当前状态中存在可继续的聊天线程，系统展示“继续上次对话”入口；用户点击后进入该 AVA 聊天。
7. 系统展示“最近使用的 Agent”“我订阅的 Agent”“团队推荐的 Agent”等区域；区域中无数据时展示空状态。
8. Agent 卡片展示头像或默认图标、名称、来源标签、描述、标签或模型信息、使用次数、最近使用时间、浏览量、喜欢数和“快捷对话”按钮。
9. 用户点击“更多”时，对应横向卡片列表继续滚动。
10. 用户点击“快捷对话”时，系统按所选 Agent 创建或转换聊天线程，设置模型和 Agent 上下文，并跳转到 AVA 聊天页面。
11. 系统展示“BoardX 推荐功能”区域，包含用户研究、深度研究和实时转录入口。
12. 用户点击推荐功能按钮时，系统创建对应 AVA 聊天线程并跳转到聊天页面；实时转录入口会记录自动打开转录的状态。
13. 如果数据仍在加载，系统在对应区域展示加载占位；如果加载失败或为空，用户仍可使用可见入口进入 AI Store、创建 Agent 或会话列表。

备选流程：
- A1：用户没有可用团队时，系统引导用户创建或加入团队。
- A2：用户没有任何可用 Agent 时，系统展示空状态，并提供创建 Agent 或进入 AI Store 的入口。
- A3：用户输入搜索关键词后，系统仅保留名称、描述或标签匹配的 Agent；没有匹配项时展示空状态。
- A4：用户所在团队没有推荐 Agent 时，“团队推荐的 Agent”区域展示当前分组暂无可展示 Agent。

异常流程：
- E1：用户未登录，系统跳转登录。
- E2：当前团队信息加载失败，系统保留页面框架，并提示用户刷新或重新选择团队。
- E3：Agent 数据加载失败，系统在对应区域展示空状态或错误反馈。
- E4：用户点击快速对话但对话创建失败，系统停止加载状态，并保留用户在 Home Page。

权限与可见性：
1. 注册用户可以查看首页工作台。
2. 未登录用户应进入身份认证入口。
3. “我订阅的 Agent”仅展示当前用户在当前团队上下文下可使用的 Agent。
4. “团队推荐的 Agent”仅展示当前团队可向成员推荐或共享的 Agent。
5. 只有对当前 Agent 有使用权限的用户才能从卡片进入快速对话。

后置条件：
- 成功查看时，用户停留在 Home Page，并明确看到当前可用的 Agent 与推荐功能。
- 成功操作时，用户进入对应 AVA 对话或推荐功能流程。

不包含：
1. 不包含 AVA 对话中的消息发送和结果生成。
2. 不包含 AI Store 中创建、审核、订阅 Agent 的完整流程。
3. 不包含 Team 管理页、Room 页面、Board 编辑页面和 Admin 后台。

业务规则：
- Home Page 的 Agent 展示必须以当前用户和当前团队为上下文。
- 最近使用、我的订阅和团队推荐是不同来源的 Agent 分组，不应混合成同一个无来源列表。
- 用户不能通过 Home Page 使用自己无权访问的 Agent。
- 推荐功能入口应进入对应 AVA 工作流，而不是停留在静态说明。
