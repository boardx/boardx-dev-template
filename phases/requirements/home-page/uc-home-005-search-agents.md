Use Case 名称：
搜索 Home Page Agent

Actor：
已登录用户

目标：
用户在 Home Page 中快速找到自己想使用的 Agent。

系统边界：
BoardX / Home

前端入口：
1. Home Page 顶部搜索框。

前置条件：
- 用户已登录。
- 用户已进入 Home Page。
- 当前页面已加载或正在加载 Agent 分组。

触发条件：
用户在搜索框中输入 Agent 名称、描述或标签关键词。

主流程：
1. 用户进入 Home Page，看到 Agent Home 欢迎区和搜索框。
2. 用户在搜索框输入关键词。
3. 系统按关键词过滤当前已加载的 Agent，匹配范围包括 Agent 名称、描述和标签。
4. 系统在“我订阅的 Agent”“团队推荐的 Agent”等分组中只展示匹配卡片，并更新分组计数。
5. 如果分组没有匹配项，系统在该分组展示空状态。
6. 用户清空搜索框后，系统恢复展示当前团队上下文下已加载的可见 Agent。
7. 用户可点击匹配 Agent 的“快捷对话”进入带有该 Agent 上下文的 AVA 聊天。

备选流程：
- A1：没有匹配结果时，系统在对应分组展示空状态。
- A2：用户只输入部分关键词时，系统仍按可识别文本进行过滤。

异常流程：
- E1：Agent 数据仍在加载时，系统展示加载占位，搜索结果随数据返回后刷新。
- E2：Agent 数据加载失败时，系统展示空状态或错误反馈。

权限与可见性：
1. 搜索只作用于用户已经有权查看的 Agent。
2. 搜索结果不得暴露当前用户无权访问的团队或私有 Agent。

后置条件：
- 用户找到并打开目标 Agent，或停留在搜索结果状态。

不包含：
1. 不包含 AI Store 全局搜索。
2. 不包含 Agent 创建、编辑、审核。

业务规则：
- 搜索范围应覆盖 Home Page 当前可见 Agent 的名称、描述和标签。
- 搜索结果必须保持原有来源分组，避免用户误判 Agent 来源。
