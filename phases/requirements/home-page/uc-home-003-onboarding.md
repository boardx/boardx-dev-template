Use Case 名称：
首页新用户引导

Actor：
新注册用户、首次进入团队的用户

目标：
系统帮助用户理解 Home Page 当前可见入口，并引导其进入 Agent、AI Store 或 AVA 推荐功能。

系统边界：
BoardX / Home

前端入口：
1. Home 页面或 Team Home 页面中的新手引导区域。
2. Board 页面中的教程或引导入口。

前置条件：
- 用户已登录。
- 用户缺少部分初始资源，例如团队、房间或白板。

触发条件：
用户首次进入首页或团队工作台。

主流程：
1. 用户首次进入 Home Page 时，系统展示 Agent Home 的欢迎区、Agent 分组、空状态和推荐功能入口。
2. 如果用户没有可展示 Agent，系统在分组中展示空状态，并提供进入 AI Store 或创建 Agent 的入口。
3. 用户查看“我订阅的 Agent”“团队推荐的 Agent”或“BoardX 推荐功能”等分组，系统展示卡片、空状态或更多入口。
4. 用户点击 Agent 卡片中的快捷对话，系统进入 AVA 对话创建或打开流程。
5. 用户点击推荐功能，例如用户研究、深度研究或实时转录，系统创建对应 AVA 场景或进入相应功能入口。
6. 用户点击 AI Store 或更多入口，系统进入对应浏览页面。
7. 如果页面或团队上下文仍在加载，系统先展示加载状态；加载完成后再展示可操作入口。
8. Home Page 本身不确认创建团队、房间或白板的完整引导流程；这些流程由对应模块完成。

备选流程：
- A1：用户跳过教程，系统关闭引导。
- A2：用户稍后再次打开教程。

异常流程：
- E1：创建资源失败，系统提示重试。
- E2：用户无权限创建资源，系统显示可用入口。

权限与可见性：
1. 注册用户可以查看首页工作台。
2. 未登录用户应进入身份认证入口。

后置条件：
- 成功时，用户完成至少一个核心启动动作。

不包含：
1. 不包含 Board 内部编辑。
2. 不包含 Team 设置管理。
3. 不包含 Admin 后台。

业务规则：
- 引导不应阻塞已有资源用户的正常操作。
- 空状态应提供清晰的下一步行动。
- Home 页面未展示完整 onboarding 时，本用例只覆盖首页可见的新手提示、推荐功能和继续入口；Board 内教程归属 Board Welcome Guide 用例。
