Use Case 名称：
从 Home Page 继续上次对话

Actor：
已登录用户

目标：
用户从 Home Page 快速回到上一次正在使用的 AVA 对话。

系统边界：
BoardX / Home

前端入口：
1. Home Page 顶部欢迎区域中的“继续上次对话”入口。

前置条件：
- 用户已登录。
- 用户存在可继续的当前或最近 AVA 对话。

触发条件：
用户点击“继续上次对话”。

主流程：
1. 用户进入 Home Page。
2. 系统展示 Agent Home 顶部区域、Agent 分组和推荐功能。
3. 系统读取当前 AVA 状态中的当前聊天线程。
4. 如果当前线程存在有效 id，系统展示“继续上次对话”入口。
5. 用户点击该入口，系统跳转到对应 AVA 聊天页面。
6. AVA 页面加载目标线程，展示消息列表、输入框、Agent 选择或当前线程标题。
7. 如果没有当前聊天线程，系统不展示该入口；用户可从 Agent 卡片、推荐功能或会话列表开始新的对话。

备选流程：
- A1：用户没有可继续的对话时，系统不展示该入口。
- A2：用户可以改为从 Agent 卡片或推荐功能创建新的 AVA 对话。

异常流程：
- E1：对应对话已不存在，系统提示无法打开，并停留在 Home Page。
- E2：用户失去该对话访问权限，系统拒绝打开。

权限与可见性：
1. 只有当前用户有权访问的 AVA 对话才能作为继续入口展示。
2. 团队切换后，继续入口应以当前团队上下文下可访问的对话为准。

后置条件：
- 成功时，用户进入目标 AVA 对话。
- 没有可继续对话时，Home Page 不展示继续入口。

不包含：
1. 不包含 AVA 对话中的消息处理细节。
2. 不包含对话列表管理。

业务规则：
- 继续入口不得打开其它团队或其它用户的对话。
- 当前团队变化后，继续入口需要随上下文刷新。
