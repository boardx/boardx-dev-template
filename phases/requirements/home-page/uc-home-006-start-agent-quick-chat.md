Use Case 名称：
从 Home Page 发起 Agent 快速对话

Actor：
已登录用户

目标：
用户从 Home Page 选择一个可用 Agent，并立即进入带有该 Agent 上下文的 AVA 对话。

系统边界：
BoardX / Home

前端入口：
1. 最近使用的 Agent 卡片。
2. 我订阅的 Agent 卡片。
3. 团队推荐的 Agent 卡片。

前置条件：
- 用户已登录。
- 用户已进入 Home Page。
- 当前 Agent 对用户可见且可使用。

触发条件：
用户点击 Agent 卡片上的“快速对话”。

主流程：
1. 用户在 Home Page 的 Agent 分组或最近使用区域浏览 Agent 卡片。
2. 系统在卡片上展示 Agent 名称、描述、来源标签、能力标签或模型、使用次数、最近使用时间、浏览量、喜欢数和“快捷对话”按钮。
3. 用户点击某个 Agent 的“快捷对话”。
4. 系统显示启动中的状态，并防止重复点击。
5. 系统读取该 Agent 的配置，判断是否需要持久化 Agent 选择、是否启用 Deep Agent，以及应使用 auto、Agent 指定模型或默认模型。
6. 系统创建新的 AVA 聊天线程，写入线程名称、Agent、模型和研究类型等当前聊天状态。
7. 系统将虚拟线程转换为真实线程后，跳转到 AVA 聊天页面。
8. 用户在聊天页面继续输入问题或任务。

备选流程：
- A1：用户从“最近使用的 Agent”进入时，系统按该最近记录对应的来源继续打开。
- A2：用户从“团队推荐的 Agent”进入时，系统以团队推荐来源打开，但仍使用当前用户身份发起对话。

异常流程：
- E1：Agent 已被取消订阅或无权访问，系统提示当前 Agent 不可用，并停留在 Home Page。
- E2：对话创建失败，系统取消启动状态，并允许用户重试。
- E3：Agent 配置加载失败，系统提示无法打开该 Agent。

权限与可见性：
1. 用户只能发起自己有权使用的 Agent 对话。
2. 团队推荐 Agent 对团队成员可见时，成员可以从 Home Page 发起对话。
3. 私有或未授权 Agent 不应出现在可点击列表中。

后置条件：
- 成功时，用户进入 AVA 对话页面。
- 失败时，用户仍停留在 Home Page，所选 Agent 不应被错误标记为已启动。

不包含：
1. 不包含对话消息生成的完整流程。
2. 不包含 Agent 的创建、编辑、发布和审核。

业务规则：
- 快速对话应保留 Agent 身份、模型能力和当前团队上下文。
- 同一时间只能启动一个 Agent 对话，避免重复创建。
