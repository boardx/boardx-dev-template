Use Case 名称：
从 Home Page 启动 BoardX 推荐功能

Actor：
已登录用户

目标：
用户从 Home Page 直接启动 BoardX 推荐的高频 AI 工作流。

系统边界：
BoardX / Home

前端入口：
1. Home Page 的 BoardX 推荐功能区域。

前置条件：
- 用户已登录。
- 用户已进入 Home Page。
- 用户所在团队允许使用 AVA。

触发条件：
用户点击推荐功能卡片上的操作按钮。

主流程：
1. 用户在 Home Page 查看“BoardX 推荐功能”区域。
2. 系统展示用户研究、深度研究和实时转录三类推荐功能卡片。
3. 每张卡片展示功能名称、说明、辅助说明、标签和操作按钮。
4. 用户点击“用户研究”的创建入口，系统创建 AVA 聊天线程，设置研究类型为 user_research，并进入聊天页面。
5. 用户点击“深度研究”的开始入口，系统创建 AVA 聊天线程，设置研究类型为 deep_research，并进入聊天页面。
6. 用户点击“实时转录”的开始入口，系统创建普通 AVA 聊天线程，记录自动打开实时转录的状态，并进入聊天页面。
7. 创建过程中按钮展示加载状态；创建失败时系统停止加载，用户仍停留在 Home Page。

备选流程：
- A1：用户只浏览推荐功能但不点击，系统保持 Home Page 当前状态。
- A2：推荐功能启动后，用户在 AVA 页面继续完成具体任务。

异常流程：
- E1：当前团队或用户信息缺失，系统不启动推荐功能，并保留在 Home Page。
- E2：创建 AVA 对话失败，系统停止加载状态，并允许用户重试。
- E3：实时转录需要麦克风权限时，后续由 AVA 对话中的转录流程处理授权。

权限与可见性：
1. 已登录用户在有可用团队上下文时可以看到推荐功能。
2. 推荐功能的最终使用仍受 AVA、团队和 Credit 相关规则约束。

后置条件：
- 成功时，用户进入对应 AVA 工作流。
- 失败时，用户停留在 Home Page。

不包含：
1. 不包含用户研究、深度研究、实时转录在 AVA 页面内的完整执行流程。
2. 不包含 Credit 扣减细节。

业务规则：
- 推荐功能必须进入可继续工作的 AVA 流程，而不是只展示静态内容。
- 同一时间只能启动一个推荐功能流程。
