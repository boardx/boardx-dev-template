Use Case 名称：
切换界面语言

Actor：
访客、注册用户

目标：
用户切换 BoardX 界面语言，以使用自己熟悉的语言浏览和操作系统。

系统边界：
BoardX / Common / Language

前端入口：
1. 页面导航或用户菜单中的语言入口。
2. 支持语言路径的页面。

前置条件：
用户正在访问 BoardX 页面。

触发条件：
用户打开语言选择入口并选择目标语言。

主流程：
1. 用户打开用户菜单，看到语言区域。
2. 系统展示当前语言，并列出 English 和中文两个可选项。
3. 当前语言对应项以选中状态展示。
4. 用户点击另一个语言。
5. 系统保存语言选择，并把当前路径中的语言前缀替换为目标语言；如果当前路径没有语言前缀，系统补上目标语言前缀。
6. 页面按新语言地址重新打开。
7. 用户继续在对应业务页面操作，看到菜单、按钮、表单提示和状态文案按目标语言展示。
8. 登录状态和当前业务上下文由应用状态和目标页面继续加载，不在语言菜单中单独创建新资源。

备选流程：
- A1：用户取消选择，系统保持当前语言。
- A2：目标语言缺少部分翻译，系统使用默认语言文案兜底。

异常流程：
- E1：语言资源加载失败，系统保留当前语言并提示稍后重试。

权限与可见性：
1. 访客和注册用户均可切换语言。
2. 语言切换不改变用户身份、权限或资源可见性。

后置条件：
系统记录或保持用户选择的语言，并在后续页面中应用。

不包含：
1. 不包含翻译内容维护。
2. 不包含浏览器语言检测算法。

业务规则：
1. 语言切换不能丢失当前页面中的资源上下文。
2. 无翻译文案必须有可理解的兜底显示。
3. 用户菜单语言选项为 English 和中文，其它语言不写入主流程。
