Use Case 名称：
切换界面主题

Actor：
注册用户、访客

目标：
用户在浅色、深色或系统主题之间切换，以获得合适的视觉体验。

系统边界：
BoardX / Common / Theme

前端入口：
1. 用户菜单中的主题入口。
2. 页面工具区中的主题切换入口。

前置条件：
用户正在访问支持主题切换的页面。

触发条件：
用户点击主题切换入口。

主流程：
1. 用户打开用户菜单，并在启用主题切换时看到主题区域。
2. 系统展示浅色、深色和系统三种主题模式。
3. 当前主题模式以选中状态展示。
4. 用户点击目标主题。
5. 系统通过主题服务更新当前主题，同时把选择写入本地存储并同步到应用状态。
6. 页面背景、文字、边框、菜单和弹窗等界面样式按新主题更新。
7. 当前页面、输入内容和业务上下文保持不变。
8. 用户刷新或重新进入页面时，系统继续应用已保存的主题模式；选择系统模式时跟随系统主题。

备选流程：
- A1：用户选择跟随系统，系统根据设备主题自动变化。
- A2：用户刷新页面，系统保持上次选择。

异常流程：
- E1：主题偏好无法保存，系统仍应用当前会话主题并提示偏好保存失败。

权限与可见性：
1. 访客和注册用户均可切换主题。
2. 主题切换不改变任何业务权限。

后置条件：
界面使用用户选择的主题模式。

不包含：
1. 不包含主题视觉规范设计。
2. 不包含具体样式规则。

业务规则：
1. 主题切换不能影响当前页面业务状态。
2. 主题模式应在页面刷新后保持一致。
