Use Case 名称：
提交带附件反馈

Actor：
注册用户、访客

目标：
用户提交产品反馈，并可附带截图或图片帮助说明问题。

系统边界：
BoardX / Feedback

前端入口：
1. 页面中的反馈入口。
2. 反馈弹窗或反馈表单中的上传入口。

前置条件：
用户正在使用 BoardX，或访问到反馈入口。

触发条件：
用户打开反馈入口并提交反馈内容。

主流程：
1. 用户点击 BoardX 页面中的反馈入口。
2. 系统打开反馈弹窗，并把应用模式切换为 feedback。
3. 系统展示“提交反馈”卡片、反馈内容输入区、上传附件入口、“提交”和“取消”按钮。
4. 用户在输入区填写反馈内容。
5. 用户点击上传入口选择附件；系统保留已选择文件，并允许在提交前移除或重新选择。
6. 用户点击提交。
7. 系统校验反馈内容不能为空；为空时展示提交错误提示，弹窗保持打开。
8. 校验通过后，提交按钮显示“提交中...”并禁用重复提交。
9. 系统把附件转换为 Base64，并随反馈内容、浏览器 user agent 和当前用户信息一起提交。
10. 提交成功后，系统清空当前白板的反馈草稿，关闭弹窗并展示“提交成功”。
11. 提交失败时，系统展示提交错误提示，用户可继续修改后重试。

备选流程：
- A1：用户不上传附件，仅提交文字反馈。
- A2：用户取消提交，系统关闭反馈表单。

异常流程：
- E1：反馈内容为空，系统阻止提交。
- E2：附件读取或提交失败，系统提示用户重试或移除附件。
- E3：提交失败，系统保留用户输入并提示稍后重试。

权限与可见性：
1. 注册用户提交反馈时可带上账号上下文。
2. 反馈提交会带上登录用户上下文；未登录用户不作为本用例主流程，未登录状态下系统应引导登录或隐藏提交入口。

后置条件：
反馈被记录，用户得到提交结果。

不包含：
1. 不包含后台处理反馈流程。
2. 不包含客服回复流程。

业务规则：
1. 反馈内容必须非空。
2. 上传入口接受图片附件；其它文件类型不作为本用例主流程，入口展示时才允许选择并提交。
3. 提交期间应防止重复提交。
