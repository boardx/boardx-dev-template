Use Case 名称：
创建连接线

Actor：
Board Owner、Board Admin、Board Member

目标：
让用户创建直线、箭头或连接对象的线。

系统边界：
BoardX / Board / Board Menu

前端入口：
1. 用户进入可编辑白板后，在画布边缘看到 Board Menu 工具栏。
2. 当前 Board Menu 直接渲染的入口包括选择、平移、便利贴、手绘、文本、连接线、形状、资源和模板。
3. 当前 Board Menu 未直接渲染文件上传、链接、图表或 AI 助手按钮；这些能力只有在其它可见入口明确展示时，才属于用户可操作能力。

前置条件：
1. 用户已打开有编辑权限的白板。
2. Board Menu 没有被系统、演示状态或只读权限隐藏。

触发条件：
用户点击、拖拽或使用快捷键进入“创建连接线”相关工具。

主流程：
1. 用户在 Board Menu 中看到连接线或箭头连接线入口。
2. 用户点击连接线入口，系统高亮连接线工具，并进入连接线创建模式。
3. 用户从画布空白处拖拽到目标位置，系统显示起点、终点和路径预览。
4. 用户从对象锚点拖拽到另一个对象时，系统高亮可连接目标，并显示连接反馈。
5. 用户松开鼠标或触控点后，系统创建连接线，并保留自由端点或对象连接关系。
6. 用户选中连接线时，系统显示端点控制点和连接线 Widget Menu。
7. 用户可以调整线宽、连接线形态、端点箭头等当前可见样式；其它样式入口只有实际显示时才可操作。
8. 用户切换工具、按 Esc 或点击选择工具时，系统退出连接线创建模式。

备选流程：
1. 用户按 Esc、点击选择工具或切换到其他工具时，系统退出当前创建模式。
2. 工具入口不可用时，系统隐藏、禁用或提示当前无法使用。
3. Visitor 和 public visitor 不显示会造成内容变更的完整创建工具。

异常流程：
1. 创建、上传或生成失败时，系统保留已有内容并显示失败反馈。
2. 权限在操作中变化时，系统停止写入并提示无权限。
3. 用户拖拽到不可用区域时，系统不创建内容或回到可放置状态。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 画布出现新内容、模式被切换，或系统明确告知该入口当前不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 只把当前界面实际显示且对当前角色可用的工具视为可操作入口。
2. 创建类工具只对可编辑角色开放。
3. 工具切换必须给出选中、高亮、光标、预览、进度或失败反馈。
