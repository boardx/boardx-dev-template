Use Case 名称：
使用 Board AI 助手

Actor：
Board Owner、Board Admin、Board Member

目标：
让用户从 Board 画布中的 AI 浮动入口打开白板级 AI 对话能力。

系统边界：
BoardX / Board / Board AI Chat

前端入口：
1. 用户进入 Board 后，系统在画布上显示 Chat AI 浮动按钮。
2. AI 聊天侧栏未打开且当前不在演示模式时，浮动按钮可见。
3. 用户点击浮动按钮后，系统打开 Board Chat Sidebar。

前置条件：
1. 用户已打开目标 Board。
2. Board 内容已经加载完成。
3. 当前不是演示模式，且 Board Chat Sidebar 未处于打开状态。

触发条件：
用户点击画布上的 Chat AI 浮动按钮。

主流程：
1. 用户在 Board 画布上看到 Chat AI 浮动按钮，按钮图标显示为 AI 头像。
2. 用户悬停按钮时，系统显示 BoardX AI Assistant 提示。
3. 用户点击按钮，系统打开 Board Chat Sidebar，并隐藏浮动按钮。
4. 系统在侧栏中显示聊天区域 Header、聊天内容区和消息输入区。
5. 用户可在 Header 中打开聊天列表、切换或新建 Board 聊天线程，并可关闭侧栏。
6. 用户在消息输入区输入问题、指令或上下文请求。
7. 用户发送消息后，系统在聊天内容区显示用户消息、AI 回复加载状态和 AI 返回内容。
8. 用户拖拽侧栏左侧调整宽度时，系统实时改变侧栏宽度，并保持画布可见。
9. 用户关闭侧栏后，系统收起 Board Chat Sidebar，并重新显示 Chat AI 浮动按钮。
10. 如果 Credit 不足、AI 处理失败或权限不足，系统显示对应提示，不覆盖原画布内容。

备选流程：
1. 已打开 Board Chat Sidebar 时，系统不再显示 Chat AI 浮动按钮。
2. 进入演示模式时，系统隐藏 Chat AI 浮动按钮。
3. 用户没有编辑权限时，可以按访问策略使用查看或对话能力，但不能通过 AI 结果直接改写 Board 内容。

异常流程：
1. 聊天线程加载失败时，系统保留侧栏并展示失败反馈。
2. 消息发送失败时，系统保留用户输入或失败消息，并允许用户重试。
3. 权限在操作中变化时，系统停止会改变 Board 内容的动作并提示无权限。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. Board Chat Sidebar 被打开、关闭、调整宽度，或聊天消息状态被更新。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Board AI 助手入口是画布浮动按钮，不是 Board Menu 直接入口。
2. 演示模式或侧栏已打开时，不显示浮动按钮。
3. AI 对话不能绕过 Board、Room 或 Team 权限直接改写画布内容。
