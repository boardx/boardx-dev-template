Use Case 名称：
调整组件样式

Actor：
Board Owner、Board Admin、Board Member

目标：
让用户修改组件颜色、边框、背景、线宽或透明度等样式。

系统边界：
BoardX / Board / Widget Menu

前端入口：
1. 用户在画布上选中单个组件或框选多个组件后，选中框附近出现悬浮 Widget Menu。
2. 菜单按对象类型显示有限入口：文本/便利贴可显示字号、字体、字重、对齐、背景色、文字色、格式应用、切换便利贴类型；连接线可显示颜色、线宽、端点和直线/曲线；图片可显示裁剪；文件可显示文件名、下载，音频文件可显示转文本。
3. 多数可编辑对象可显示锁定、删除和 AI 助手；锁定对象通常只保留锁定状态入口。

前置条件：
1. 用户已打开有编辑权限的白板。
2. 至少一个组件处于选中状态。

触发条件：
用户在 Widget Menu 中点击“调整组件样式”相关入口。

主流程：
1. 用户选中文本、便利贴、形状、连接线或其它支持样式的对象，系统显示对象适用的样式入口。
2. 用户打开颜色入口，系统显示当前可用色板或颜色选择器。
3. 用户选择背景色、文字色、边框色或连接线颜色，系统即时更新对象外观。
4. 用户打开线宽、边框或端点入口，系统展示当前可见选项。
5. 用户选择线宽、边框粗细、箭头端点或路径形态，系统更新对象显示。
6. 用户打开文本样式入口并选择字号、字体、字重或对齐方式，系统更新对象内文字。
7. 多选对象时，系统只显示共同适用的样式项；不同值可以显示混合状态。
8. 锁定、只读或无权限对象不允许通过样式入口被修改。

备选流程：
1. 多选时，系统只展示多选适用的锁定、删除、AI、对齐或整理类入口。
2. 当前组件类型不支持的动作不显示。
3. 对象被锁定时，系统隐藏会改变内容的入口，或要求先解锁。

异常流程：
1. 权限不足时，系统不执行修改并提示无权限。
2. 转换、刷新、下载、裁剪或 AI 处理失败时，系统保留原组件并显示失败反馈。
3. 组件在操作期间被协作者删除或权限改变时，系统关闭菜单并刷新状态。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 组件属性、内容、锁定状态或文件处理结果与用户动作一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Widget Menu 不是全量工具箱，显示项必须由对象类型、选中数量、锁定状态和角色权限共同决定。
2. 不能因为某个菜单组件存在就声明所有对象都显示该入口。
3. Visitor 和 public visitor 不使用会改变组件的菜单动作。
