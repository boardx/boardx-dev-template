Use Case 名称：
使用对齐参考线

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成使用对齐参考线。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到画布上的已有组件。
2. 用户选中并拖动组件时，画布显示选中框和移动反馈。
3. 当组件靠近可对齐位置时，画布可显示对齐参考线或吸附反馈。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户拖动或调整组件，准备与其它组件或画布参考位置对齐。

主流程：
1. 用户点击一个可编辑组件，系统显示选中框和控制点。
2. 用户拖动组件，系统让组件跟随指针移动。
3. 组件接近其它对象的边缘、中心或可对齐位置时，系统显示参考线或吸附提示。
4. 用户继续拖动，系统持续更新组件位置和参考线状态。
5. 用户释放鼠标或触控操作，系统把组件停留在当前位置。
6. 如果触发吸附，组件停在对齐后的位置；如果没有触发吸附，组件停在用户释放的位置。
7. 操作结束后，系统保留选中状态，并在协作场景中让其他在线用户看到最终位置。

备选流程：
1. 用户拖动多个已选组件时，系统按多选整体显示移动和对齐反馈。
2. 用户按 Esc 或点击空白处时，系统退出当前选择状态。
3. 对象被锁定时，系统限制移动，不显示可编辑的拖动结果。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 组件位置与用户释放位置或参考线吸附位置一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
