Use Case 名称：
缩放画布并使用缩放控制条

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中通过右下角缩放控制条缩放画布和适应内容。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后，在画布右下角看到缩放控制条；当前界面没有展示可拖拽定位的缩略画布小地图。
2. 缩放控制条包含适应屏幕、缩小、当前百分比下拉和放大入口。
3. 百分比下拉展示固定缩放值 50%、70%、100%、150%、200%，并提供适应屏幕入口。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要放大查看细节、缩小查看整体，或让画布自动适配当前内容。

主流程：
1. 用户查看右下角缩放控制条，系统显示当前缩放百分比。
2. 用户点击放大或缩小，系统以当前视角中心缩放画布，并立即更新百分比。
3. 用户打开百分比下拉，系统显示可选缩放值和适应屏幕动作。
4. 用户选择 50%、70%、100%、150% 或 200%，系统切换到对应比例，画布内容按新比例显示。
5. 用户点击适应屏幕，系统把画布视角调整到能看到当前内容的范围，并更新百分比。
6. 用户继续平移或缩放时，系统保持右下角百分比与画布视角一致。
7. 缩放结束后，用户仍停留在同一 Board，已有选中对象、在线头像和协作状态继续可见。

备选流程：
1. 用户通过触控板或鼠标滚轮缩放时，系统同样更新视角和百分比。
2. 用户选择 100% 时，系统恢复到标准比例。
3. 演示模式下隐藏缩放控制时，用户只能使用当前模式允许的导航能力。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 缩放比例低于或高于系统允许范围时，系统停在可用边界，不继续缩放。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 画布视角和缩放百分比与用户选择一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
4. 不能把组件名或容器 id 中的 MiniMap 写成用户可见的缩略画布功能；当前可确认的是缩放控制条。
