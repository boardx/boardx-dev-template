Use Case 名称：
选择和多选组件

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成选择和多选组件。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到画布上的便利贴、形状、连接线、文本、图片、文件、手绘或图表等组件。
2. 用户点击单个组件时看到选中框、控制点和对应 Widget Menu。
3. 用户框选或多选组件时看到整体选中边界和多选可用菜单。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要查看、移动、对齐、批量管理或编辑一个或多个组件。

主流程：
1. 用户点击画布上的单个组件，系统显示该组件的选中框和控制点。
2. 系统根据组件类型显示可用 Widget Menu，例如文本样式、颜色、线宽、裁剪、下载、锁定或连接线端点。
3. 用户拖动选中组件，系统移动组件并显示位置变化。
4. 用户拖动控制点，系统按组件类型调整大小、路径、显示区域或文字排版区域。
5. 用户在空白区域拖出选择框，系统把框内可选组件加入多选，并显示整体选中边界。
6. 用户对多选对象执行移动、对齐、分布、编组、锁定或删除，系统只展示多选适用动作并更新结果。
7. 用户点击空白处或按 Esc，系统取消选择并隐藏相关菜单。
8. 操作结束后，系统保留最终组件位置、尺寸、选中状态或多选状态。

备选流程：
1. 用户多选中包含不同组件类型时，系统只展示这些组件共同支持的动作。
2. 对象被锁定时，系统可显示其存在，但限制移动、缩放、改内容和删除。
3. 只读用户可以选择或查看允许范围内的内容，但不能保存编辑结果。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 单选或多选状态、组件位置、尺寸和菜单可见性与用户操作一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
