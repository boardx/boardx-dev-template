Use Case 名称：
用键盘操作组件

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成用键盘操作组件。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到画布、主工具栏和已有组件。
2. 用户选中组件后看到选中框、控制点和 Widget Menu。
3. 用户可以使用键盘切换工具、临时平移、取消操作或打开上下文菜单。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要不用鼠标完成工具切换、临时平移、取消编辑或管理选中组件。

主流程：
1. 用户按 V，系统切换到选择工具，画布允许点击、框选和移动组件。
2. 用户按 T、D、S、N 或 L，系统分别切换到文本、手绘、形状、便利贴或连接线工具；主工具栏状态同步变化。
3. 用户按 C 时，系统切换到图表模式；由于 Board Menu 当前不直接渲染图表按钮，系统只更新可感知的工具状态或后续创建结果，不把图表按钮显示作为前置条件。
4. 用户按住空格，系统临时进入平移状态，光标变为抓手，拖动画布时只改变视角。
5. 用户松开空格，系统退出临时平移，恢复到之前的工具。
6. 用户按 Esc，系统取消当前工具或编辑状态，并回到选择状态。
7. 用户在选中组件时按上下文菜单键，系统在选中对象附近打开 Context Menu。
8. 用户从菜单执行复制、删除、锁定等动作后，系统更新组件状态并给出可见反馈。

备选流程：
1. 用户正在输入文本时，系统把字母键当作文本输入，不触发工具切换。
2. 对象被锁定时，键盘动作不能移动、编辑或删除该对象。
3. 只读用户可以使用导航类快捷键，但不能通过快捷键创建或修改内容。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 当前工具、视角、菜单状态或组件状态与用户键盘操作一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
