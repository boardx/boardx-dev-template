Use Case 名称：
管理 Board 可见范围

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
让具备管理权限的用户设置谁可以访问 Board。

系统边界：
BoardX / Board / Share

前端入口：
1. 用户在 Board Header 点击“分享白板”入口。
2. 分享面板展示标题、访问范围下拉、复制链接按钮和显示/隐藏二维码入口。
3. 访问范围下拉可显示公开、团队成员可访问、房间成员可访问等与当前 Board 所属空间匹配的选项。
4. 当前用户不是 Room Owner 或 Room Admin 时，访问范围下拉保持禁用，但复制链接和二维码入口仍显示。

前置条件：
1. 用户已打开 Board。
2. 系统已加载 Board 当前访问级别和用户在 Room/Team 中的角色。

触发条件：
用户打开分享面板并尝试调整访问范围。

主流程：
1. 用户点击“分享白板”，系统打开分享面板。
2. 系统显示当前访问范围下拉，并按 Board 所属 Team 或 Room 展示可选范围。
3. 当前用户是 Room Owner 或 Room Admin 时，可以选择新的访问范围，系统立即更新面板中的当前选择。
4. 系统保存访问范围，并把变更同步给当前 Board。
5. 用户点击复制链接，系统复制包含当前 Board 地址和视角参数的链接，并提示复制成功或失败。
6. 用户点击显示二维码，系统在面板中生成二维码；再次点击后隐藏二维码。
7. 用户关闭分享面板后，Board 继续按最新访问范围控制后续访问。

备选流程：
1. 非 Room Owner 或 Room Admin 用户仍可复制链接或显示二维码，但访问范围选择保持禁用。
2. 目标 Board 不属于 Team 或 Room 时，系统不展示对应范围。
3. 用户只需要二维码时，可以不改变访问范围，直接显示二维码。

异常流程：
1. 保存失败时，系统恢复或保留原访问范围并提示失败。
2. 权限不足时，系统提示无权限。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. Board 访问范围与用户选择一致，分享链接遵循新的访问策略。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 公开范围允许 public visitor 查看，不自动授予编辑权限。
2. Room/Team 范围要求访问者具备对应成员身份。
3. 访问范围变更应向在线协作者同步。
