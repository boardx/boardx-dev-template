Use Case 名称：
通过公开链接加入或查看 Board

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
让用户通过公开链接进入允许访问的 Board，并在需要时加入协作身份。

系统边界：
BoardX / Board / Public Access

前端入口：
1. 用户收到从分享面板复制的公开链接或二维码。
2. 链接可包含 Board 地址和当前画布视角参数。
3. 用户打开后看到 Board 加载状态、可查看内容、在线状态和必要的登录/加入提示。

前置条件：
1. Board 的访问范围允许公开查看。

触发条件：
用户打开公开 Board 链接。

主流程：
1. 用户在浏览器打开公开链接。
2. 系统显示 Board 加载状态，并校验公开访问策略。
3. 系统加载成功后展示 Board 标题、画布内容、同步状态和允许查看的 Header 入口。
4. 链接带有视角参数时，系统打开到对应画布视角。
5. public visitor 可以平移、缩放、查看组件内容、同步状态和在线状态。
6. public visitor 尝试创建、编辑、删除、改变分享范围或执行成员级操作时，系统阻止并提示登录、加入空间或无权限。
7. 用户登录后，系统按其 Team、Room 或 Board 身份重新判断 Member/Visitor 权限，并刷新可见入口。

备选流程：
1. Board 已改为非公开时，系统展示无权限或登录/申请访问入口。
2. 链接带有视角参数时，系统打开到对应画布视角。

异常流程：
1. Board 不存在、已删除或无权访问时，系统提示并提供返回入口。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 用户以公开查看身份进入 Board，或完成登录后按成员身份继续。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 公开访问是查看策略，不是编辑授权。
2. public visitor 不能通过链接看到成员级菜单或管理入口。
