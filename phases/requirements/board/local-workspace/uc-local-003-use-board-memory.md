Use Case 名称：
使用 Board Memory

Actor：
Board Owner、Board Admin、Board Member

目标：
在本地工作区中使用 Board Memory。

系统边界：
BoardX / Local Workspace

前端入口：
1. 用户在 Board 内打开 Board Memory 或包含 Board Memory 的记忆面板。
2. 系统显示 Board Memory 标题、说明、记忆数量、搜索框、记忆列表、添加记忆输入框和添加按钮。
3. 每条记忆以可阅读的条目展示，鼠标悬停或聚焦时可看到删除入口。

前置条件：
1. 用户已登录并打开可编辑 Board。
2. Board Memory 入口已显示，并且用户有权查看或维护该 Board 的记忆。

触发条件：
用户打开 Board Memory 或包含 Board Memory 的记忆面板。

主流程：
1. 用户打开 Board Memory，系统显示已有记忆列表；没有记忆时显示空状态。
2. 用户在搜索框输入关键词，系统过滤记忆列表；没有匹配结果时显示无结果提示。
3. 用户在输入框填写新的白板相关记忆，系统允许使用 Shift + Enter 换行。
4. 用户点击“添加记忆”或在输入框按 Enter，系统显示保存中状态，并在保存成功后把新记忆加入列表和数量统计；Shift + Enter 只在输入框中换行。
5. 用户在记忆条目上点击删除入口，系统打开删除确认。
6. 用户确认删除，系统显示保存中状态，并在成功后从列表移除该记忆。
7. 保存或删除失败时，系统保留原内容并提示失败，用户可以重试。

备选流程：
1. 搜索框为空时，系统展示全部记忆。
2. 用户输入为空或只包含空白时，添加按钮保持不可用。
3. 用户取消删除确认时，系统保留原记忆。
4. 能力未开启时，系统不展示 Board Memory 入口或提示当前不可用。

异常流程：
1. 读取、保存或删除失败时，系统保留用户输入或原记忆列表，并提示失败原因。
2. 敏感内容或无权限内容不能被加入记忆。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 用户完成一次 Board Memory 查看、搜索、添加或删除，并能看到最终列表状态。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Local Workspace 能力由当前环境显示的入口决定；未展示入口时，用户不能从 Board 执行对应本地能力。
2. Board Memory 应区分用户主动保存和系统临时上下文。
3. 本地模型和本地工具不作为当前 Board 的已确认可见能力。
