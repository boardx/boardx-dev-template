Use Case 名称：
使用 Board Chat

Actor：
Board Owner、Board Admin、Board Member

目标：
在 Board 中使用当前可见的 Board Chat；本地模型和本地工具只有入口明确展示时才纳入流程。

系统边界：
BoardX / Local Workspace

前端入口：
1. 用户在 Board 内看到 Board Chat 入口时，才能进入该场景。
2. 当前 Board 可确认入口包括 Board Chat。
3. 本地模型选择和本地工具调用不作为当前 Board 的已确认可见入口。

前置条件：
1. 用户已登录并打开可编辑 Board。
2. Board Chat 入口已显示。

触发条件：
用户打开 Board Chat 入口。

主流程：
1. 用户打开 Board Chat。
2. 系统显示聊天区域、消息输入区和可见的聊天线程操作。
3. 用户输入问题或指令。
4. 系统显示发送后处理状态，例如生成中或失败提示。
5. 系统返回回答，用户可以继续追问、复制结果，或使用当前界面实际展示的后续动作。
6. 如果界面未展示“插入 Board”或本地工具入口，不能把结果写入 Board、选择本地模型或调用本地工具写成主流程事实。
7. 用户关闭 Board Chat 后，系统回到 Board 画布，并保留当前聊天线程状态。

备选流程：
1. 能力未开启时，系统不展示入口或提示当前不可用。
2. 用户没有足够权限读取 Board 内容时，系统限制引用范围。
3. 本地模型或本地工具入口未显示时，系统不提供模型选择或本地工具调用。

异常流程：
1. 读取、生成、保存或插入失败时，系统保留用户输入并提示失败原因。
2. 敏感内容或无权限内容不能被加入记忆或发送给工具。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 用户完成一次 Board Chat 操作，并知道结果是仅显示在当前会话，还是通过当前可见动作被进一步处理。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Local Workspace 能力由当前环境显示的入口决定；未展示入口时，用户不能从 Board 执行对应本地能力。
2. Board Memory 应区分用户主动保存和系统临时上下文。
3. 本地模型和本地工具不作为当前 Board 的已确认可见能力。
