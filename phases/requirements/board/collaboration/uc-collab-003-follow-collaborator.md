Use Case 名称：
跟随协作者视角

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在多人场景中跟随协作者视角。

系统边界：
BoardX / Collaboration

前端入口：
1. 用户进入 Board 后，在 Header 区域看到在线成员头像和跟随相关入口。
2. 启用跟随后，画布顶部显示跟随状态条。
3. 跟随状态条显示正在被跟随的人或“成员正在跟随你”，并提供暂停、恢复和停止入口。

前置条件：
1. 多个用户同时打开同一个有访问权限的 Board。
2. 实时连接正在建立或已经建立。

触发条件：
用户进入 Board、移动光标、编辑内容或选择跟随某位协作者。

主流程：
1. 用户在在线成员或跟随入口中选择要跟随的协作者。
2. 系统把用户画布视角移动到协作者当前视角，并显示跟随状态条。
3. 协作者平移或缩放画布时，系统同步调整用户的画布视角。
4. 用户点击暂停，系统保留当前跟随状态条，但停止继续套用对方视角变化。
5. 用户点击恢复，系统继续按协作者视角更新自己的画布。
6. 用户点击停止，系统退出跟随状态，状态条消失，用户保留当前视角。
7. 如果用户是被跟随者，系统显示“成员正在跟随你”一类提示，让用户知道自己的视角正在影响他人。

备选流程：
1. 被跟随协作者离线或停止广播视角时，系统退出或停止更新跟随状态。
2. 用户手动平移或缩放时，系统可暂停或退出跟随，以用户主动视角为准。
3. Visitor 可以跟随查看视角，但不能因为跟随获得编辑能力。

异常流程：
1. 实时连接失败时，系统提示连接异常并保留只读或本地状态。
2. 权限变更后，系统重新判断是否允许继续查看或编辑。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 用户能理解谁在线、谁在操作，以及自己的视角是否正在跟随他人。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 在线状态和光标是协作反馈，不代表对方拥有相同权限。
2. 跟随只改变视角，不改变 Board 内容。
3. 协作同步不能让只读用户获得编辑能力。
