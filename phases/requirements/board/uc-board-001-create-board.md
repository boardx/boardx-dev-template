Use Case 名称：
创建 Board

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
让用户在有权限的空间内创建一个新白板。

系统边界：
BoardX / Board 列表 / Board 详情

前端入口：
1. 用户在 Room、Team、最近访问或搜索结果中看到白板卡片、搜索框、新建白板按钮、收藏入口和卡片更多操作菜单。
2. 白板卡片通常展示名称、封面或缩略图、最近编辑信息；无内容时显示空状态。
3. 用户打开白板后看到加载状态、白板标题、画布内容、Header、Board Menu、缩放/小地图等可用入口；无权限时看到提示或返回入口。

前置条件：
1. 用户已登录，或正在访问允许公开查看的白板链接。
2. 目标白板所属 Room/Team 与当前用户身份已经完成校验。

触发条件：
用户选择与“创建 Board”相关的卡片、按钮、菜单项或链接。

主流程：
1. 用户进入 Room、Team 或最近访问页面后，看到白板列表区域、已有白板卡片以及“创建白板”按钮；在移动端入口显示为加号、新建图标或当前页面提供的创建入口。
2. 用户点击“创建白板”，系统打开“创建白板”弹窗，并把焦点放到“白板名称”输入框。
3. 用户看到“白板名称”字段、默认名称提示、白板标签区域和创建按钮；名称为空时创建按钮不可用。
4. 用户输入白板名称；如果需要标签，用户在标签输入区域输入标签名并按 Enter，系统把标签显示成可点击移除的标签块。
5. 用户点击“创建”或在名称输入框中按 Enter，系统将按钮切换为“创建中”或禁用状态，避免重复提交。
6. 创建成功后，系统关闭弹窗并进入新白板页面；用户看到画布加载完成后的 Header、白板标题、Board Menu、在线成员和同步状态。
7. 创建失败时，系统停留在弹窗或列表页，恢复按钮可点击状态，并展示失败反馈；用户可以修改名称后重试或关闭弹窗。

备选流程：
1. 用户取消弹窗、关闭菜单或返回上一层时，系统不改变白板数据。
2. 搜索无结果时，系统展示空状态，并允许用户清空关键词或改用新建入口。
3. 公开访问者或只读用户尝试成员动作时，系统引导登录、申请加入或提示无权限。

异常流程：
1. 权限不足时，系统提示无权限并保留原状态。
2. 网络、保存或加载失败时，系统展示失败反馈，用户可以重试。
3. 白板已被删除、移动或权限变更时，系统刷新列表或阻止继续使用旧入口。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 白板列表、最近访问、卡片状态或当前打开的白板与操作结果一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 列表、卡片和详情页只展示当前角色可进入的操作。
2. 高风险动作需要清晰确认、处理中状态和完成/失败反馈。
3. 公开链接只代表可访问性，不代表具备编辑或管理权限。
