Use Case 名称：
打开快捷键帮助

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
让用户查看当前可用快捷键。

系统边界：
BoardX / Board / Header

前端入口：
1. 用户进入白板后，在顶部左侧看到返回、白板标题、撤销、重做、语音录制、模式切换和同步状态；窄屏下标题和左侧部分入口收起或不显示。
2. 顶部右侧显示在线成员、分享、跟随、计时器、幻灯片和更多菜单。
3. 更多菜单包含备份、快捷键、白板设置、语言切换和登出；分享面板包含访问范围、复制链接和二维码开关。

前置条件：
1. 用户已通过白板访问校验。
2. 白板基础信息、当前用户身份和 Header 状态已加载。

触发条件：
用户查看 Header 或点击与“打开快捷键帮助”相关的入口。

主流程：
1. 用户点击 Header 右侧更多菜单，并选择“快捷键”。
2. 系统关闭更多菜单并打开快捷键弹窗，标题显示快捷键相关文案。
3. 弹窗顶部展示上下文相关提示，提醒用户快捷键会随当前选择、输入焦点或画布状态变化。
4. 用户在弹窗中看到按分类排列的快捷键，例如工具、编辑、排列、视图、文本、帮助、系统和其他；每条快捷键显示操作说明和按键组合。
5. 系统根据用户平台显示对应按键样式，例如 Mac 上显示 Command、Shift 等符号，其他平台显示 Ctrl、Shift、Alt 等文字。
6. 用户滚动弹窗可以查看更多分类和快捷键；快捷键服务尚未加载完成时，系统只展示已可确认的内容或保持空白区域。
7. 用户点击关闭、按 Esc 或点击弹窗外部后，系统关闭快捷键弹窗并回到白板，画布内容不改变。

备选流程：
1. 计时器运行中时，计时器入口不可再次打开设置，按钮保持禁用或不可操作。
2. 幻灯片创建或截图过程中，幻灯片侧栏不能立即关闭，系统保持创建状态。
3. 窄屏时部分入口折叠或隐藏，但不扩大用户权限。

异常流程：
1. 权限不足时，系统隐藏、禁用或提示无权限。
2. 麦克风、复制链接、备份、导出或保存失败时，系统保留原状态并显示失败提示。
3. 同步异常时，系统展示同步中或异常反馈，避免用户误判为已保存。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. Header、画布、侧栏、弹窗或协作者状态与用户操作结果一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Header 是白板的一级状态和操作入口。
2. 状态可见不代表具备编辑或管理权限。
3. 分享范围、备份恢复、设置和登出等动作必须给出明确反馈。
