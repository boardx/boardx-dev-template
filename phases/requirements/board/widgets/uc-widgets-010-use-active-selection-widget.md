Use Case 名称：
使用多选组合状态

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中同时选择多个组件，对它们执行批量移动、对齐、编组、锁定、删除等操作。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. 画布选择工具。
2. 框选多个组件形成的多选状态。
3. 按住辅助键逐个选择多个组件形成的多选状态。
4. 多选状态下出现的 Widget Menu 或 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户框选或追加选择多个组件。

主流程：
1. 用户启用选择工具，并在画布上框选多个组件，或逐个追加选择多个组件。
2. 系统显示包含多个组件的多选边界，并标识当前已选对象集合。
3. 用户拖拽多选边界或其中可拖动区域，系统整体移动所选组件。
4. 用户拖拽多选控制点，系统按当前界面规则整体缩放或调整所选组件集合。
5. 用户点击对齐或分布入口，系统按用户选择的方向对所选组件进行左对齐、右对齐、顶对齐、底对齐、水平居中、垂直居中或分布。
6. 用户点击编组，系统把所选组件组成一个组；用户点击取消编组，系统把组恢复为可独立选择的组件。
7. 用户点击锁定或解锁，系统批量更新所选组件的锁定状态。
8. 用户点击删除，系统批量删除所选组件。
9. 用户使用复制、剪切、粘贴、置顶置底、导出选中内容或保存为模板等动作时，系统对当前选中集合执行操作。
10. 操作完成后，系统保留更新后的组件状态，并允许用户继续单选或多选。

备选流程：
1. 所选集合中包含锁定组件时，系统仅允许不会改变锁定组件的动作，或提示部分对象不可操作。
2. 所选集合中对象类型不一致时，系统只展示所有对象共同支持或多选支持的动作。
3. 用户点击空白画布时，系统退出多选状态。
4. 用户双击组内对象时，系统可进入组内选择或单对象选择状态。

异常流程：
1. 用户没有编辑权限时，系统不允许批量移动、缩放、删除、编组或改样式。
2. 批量操作过程中部分对象被其他协作者删除、锁定或权限改变时，系统跳过不可操作对象或刷新选择状态。
3. 保存为模板或导出选中内容失败时，系统保留原多选内容，并提示用户重试。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 多个组件被选中、移动、缩放、对齐、分布、编组、锁定、删除、导出或保存为模板，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 多选状态不是新的业务内容对象，而是用户对多个组件执行批量操作的临时状态。
4. 多选菜单只能展示当前选中集合在角色权限和对象状态下共同可执行的动作。
5. 锁定组件不得通过多选操作绕过锁定限制。
