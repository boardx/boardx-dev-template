Use Case 名称：
使用手绘组件

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中通过手绘笔迹表达草图、标注或自由线条，并可调整笔迹样式和管理绘制内容。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 主工具栏中的手绘或画笔入口。
2. Board 主工具栏中的橡皮擦入口。
3. 画布上已有手绘笔迹。
4. 选中手绘笔迹后出现的 Widget Menu 或 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户选择手绘工具并在画布上拖动画笔，或选中已有手绘笔迹。

主流程：
1. 用户点击手绘工具，系统进入绘制状态。
2. 用户在画布上按住并拖动，系统实时显示笔迹路径。
3. 用户松开后，系统把本次笔迹保存为画布上的手绘对象。
4. 用户选中手绘对象，系统显示边界、控制点和可用样式菜单。
5. 用户拖拽手绘对象，系统移动笔迹位置。
6. 用户拖拽控制点，系统缩放手绘对象。
7. 用户调整笔迹颜色或线宽，系统更新手绘对象的显示效果。
8. 用户使用橡皮擦擦除绘制内容时，系统移除命中的笔迹或笔迹片段，并保留其它未命中的内容。
9. 用户通过 Context Menu 对手绘对象执行复制、剪切、粘贴、删除、锁定、解锁、置顶置底等动作。
10. 操作完成后，手绘内容留在画布中，并参与协作同步、撤销重做、选择、多选和导出。

备选流程：
1. 用户取消绘制或没有形成有效笔迹时，系统不创建手绘对象。
2. 多选手绘对象时，系统进入多选状态，只展示多选适用动作。
3. 手绘对象被锁定时，系统允许查看，但不允许移动、缩放、改色、擦除或删除。

异常流程：
1. 用户没有编辑权限时，系统不进入绘制状态，也不允许擦除或修改已有笔迹。
2. 协作过程中手绘对象被其他用户删除或锁定时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 手绘对象被创建、移动、缩放、改色、改线宽、擦除、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 手绘笔迹的路径、位置、尺寸、颜色和线宽都属于用户可感知状态。
4. 橡皮擦只能删除用户有权修改的绘制内容。
