Use Case 名称：
使用 CanvasX Widgets

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 画布中创建、选择和管理 CanvasX 组件，并理解不同组件类型对应的可操作能力边界。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 主工具栏中的选择、便利贴、文本、形状、连接线、手绘、资源和模板入口；图片和图标从资源面板进入，文件入口只有实际显示时才可用。
2. 画布上已有组件。
3. 选中组件后出现的 Widget Menu。
4. 右键或更多操作打开的 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户创建新组件，或选中画布上的已有组件。

主流程：
1. 用户从主工具栏、资源面板、模板面板、拖放或粘贴入口创建组件。
2. 系统在画布中显示新组件，并根据组件类型展示文本光标、图片预览、文件状态、连接端点、手绘路径、形状边界或便利贴内容。
3. 用户点击组件，系统显示选中框、控制点和该组件类型适用的 Widget Menu。
4. 用户拖拽组件，系统移动组件位置。
5. 用户拖拽控制点，系统按组件类型改变大小、路径或显示区域。
6. 用户使用键盘移动、复制、粘贴、删除或撤销重做，系统更新选中组件状态。
7. 用户通过 Widget Menu 修改组件类型支持的属性，例如文本样式、便利贴形态、形状颜色、连接线端点、图片裁剪、文件下载或音频转文本。
8. 用户通过 Context Menu 执行通用管理动作，例如复制、剪切、粘贴、删除、锁定、解锁、置顶置底、对齐、编组、取消编组、导出选中内容或保存为模板。
9. 用户多选组件时，系统进入多选状态，并只展示多选适用的移动、对齐、分布、锁定、删除、编组等动作。
10. 操作完成后，组件保留在画布中，并参与协作同步、撤销重做、选择、多选、导出和权限控制。

备选流程：
1. 组件被锁定时，系统允许查看和受限选择，但不允许移动、缩放、改内容、改样式或删除。
2. 当前组件类型不支持某个菜单动作时，系统不显示该入口。
3. Visitor 或 Public visitor 进入 Board 时，系统只提供查看、平移、缩放和允许范围内的查看操作。
4. 图表组件在当前 Board Menu 中没有直接按钮；文件、链接等入口未在当前 Board Menu 直接渲染，未展示入口不能被当作当前可操作能力。

异常流程：
1. 用户没有编辑权限时，系统不创建组件，也不允许修改已有组件。
2. 文件处理、图片加载、AI 辅助或图表渲染失败时，系统提示失败，并保留原组件或可恢复状态。
3. 协作过程中组件被其他用户删除、锁定或权限改变时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 组件被创建、选中、移动、缩放、改样式、改内容、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 组件的可操作能力必须由组件类型、锁定状态、选中数量和角色权限共同决定。
4. 不能因为某个组件支持某动作，就把该动作默认扩展到所有组件。
5. 实现中存在的组件类型不等于当前用户一定能在界面创建。
