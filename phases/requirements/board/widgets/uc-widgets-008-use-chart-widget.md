Use Case 名称：
使用图表组件

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中查看或使用图表组件，并对图表组件进行选择、移动、锁定、删除和 AI 辅助等可用操作。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 中已有图表组件。
2. 选中图表后出现的 Widget Menu 或 Context Menu。
3. Board Menu 当前不直接渲染图表按钮；通过快捷键创建图表由 Board Menu 对应用例描述。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户选中画布上的已有图表组件。

主流程：
1. 用户点击画布上的已有图表组件。
2. 系统在画布中显示图表内容、图表边界和选中反馈。
3. 用户拖拽图表组件，系统移动图表位置。
4. 用户拖拽控制点，系统调整图表组件的显示大小。
5. 用户选中图表后，系统展示图表组件适用的菜单入口。
6. 用户可以锁定或解锁图表组件，系统更新锁定状态。
7. 用户可以删除图表组件，系统从画布移除该图表。
8. 用户点击 AI 辅助入口时，系统以图表组件或其上下文作为对象进入 AI 辅助流程。
9. 用户通过 Context Menu 对图表执行复制、剪切、粘贴、置顶置底、编组或取消编组等通用动作。
10. 操作完成后，图表组件留在画布中，并参与协作同步、撤销重做、选择、多选和导出。

备选流程：
1. 用户没有编辑权限时，系统允许查看图表内容，但不允许移动、缩放、删除或改写图表状态。
2. 多选图表和其它对象时，系统进入多选状态，只展示多选适用动作。
3. 图表被锁定时，系统允许查看，但不允许移动、缩放、删除或触发会改变图表的动作。

异常流程：
1. 图表内容加载或渲染失败时，系统展示失败反馈，并保留图表组件的可识别状态。
2. 用户没有编辑权限时，系统不创建图表，也不允许修改已有图表组件。
3. AI 辅助失败时，系统保留原图表组件。
4. 协作过程中图表被其他用户删除或锁定时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 图表被查看、移动、缩放、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 图表的可见内容、位置、尺寸、锁定状态和加载状态都属于用户可感知状态。
4. Board Menu 当前不直接渲染图表按钮；本用例只描述已有图表组件的可见操作边界。
