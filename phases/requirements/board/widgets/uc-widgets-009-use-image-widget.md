Use Case 名称：
使用图片组件

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中添加图片，并对图片的位置、尺寸、裁剪和状态进行管理。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 资源或图片入口。
2. 从本地拖放或粘贴到画布的图片。
3. 画布上已有图片组件。
4. 选中图片后出现的 Widget Menu 或 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户添加图片，或选中画布上的已有图片。

主流程：
1. 用户通过图片入口、拖放或粘贴添加图片。
2. 系统在画布中展示图片预览；加载中时展示加载状态。
3. 用户选中图片，系统显示选中框、缩放控制点和图片可用菜单。
4. 用户拖拽图片本体，系统移动图片位置。
5. 用户拖拽边角或控制点，系统改变图片显示大小。
6. 用户点击裁剪入口，系统进入图片裁剪状态，并显示裁剪边界或裁剪控制点。
7. 用户调整裁剪区域并确认，系统更新图片可见区域。
8. 用户取消裁剪，系统恢复到裁剪前的图片显示状态。
9. 用户通过 Context Menu 对图片执行复制、剪切、粘贴、删除、锁定、解锁、置顶置底、编组或取消编组等动作。
10. 用户点击 AI 辅助入口时，系统以选中图片作为上下文进入 AI 辅助流程。
11. 操作完成后，图片留在画布中，并参与协作同步、撤销重做、选择、多选和导出。

备选流程：
1. 图片仍在加载时，用户可以等待加载完成；系统不应把未加载完成的图片误显示为可裁剪完成状态。
2. 多选图片和其它对象时，系统进入多选状态，只展示多选适用动作。
3. 图片被锁定时，系统允许查看，但不允许移动、缩放、裁剪、删除或替换。

异常流程：
1. 图片加载失败时，系统展示失败反馈，并保留可删除或重试的状态。
2. 用户没有编辑权限时，系统不添加图片，也不允许修改已有图片。
3. AI 辅助失败时，系统保留原图片。
4. 协作过程中图片被其他用户删除或锁定时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 图片被添加、移动、缩放、裁剪、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 图片的来源、预览、位置、尺寸、裁剪区域和锁定状态都属于用户可感知状态。
4. 裁剪不应破坏用户对图片原始内容的可恢复预期，除非用户明确确认不可逆处理。
