Use Case 名称：
使用文件组件

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中添加文件，查看文件卡片信息，并对文件组件进行移动、下载、转写或删除等操作。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 文件或资源入口。
2. 从本地拖放到画布的文件。
3. 画布上已有文件组件。
4. 选中文件后出现的 Widget Menu 或 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户添加文件，或选中画布上的已有文件组件。

主流程：
1. 用户通过文件入口或拖放方式添加文件。
2. 系统在画布中创建文件组件，并展示文件名、文件类型或可识别的文件状态。
3. 用户选中文件组件，系统显示选中框、控制点和文件可用菜单。
4. 用户拖拽文件组件，系统移动文件卡片位置。
5. 用户拖拽控制点，系统调整文件卡片显示大小。
6. 用户点击文件名入口时，系统允许查看或编辑文件显示名称。
7. 用户点击下载入口，系统开始下载文件，并展示下载中、成功或失败反馈。
8. 用户选中音频文件并点击转文本入口，系统进入音频转文本处理流程；成功后在画布中生成或展示转写文本结果。
9. 用户通过 Context Menu 对文件组件执行复制、剪切、粘贴、删除、锁定、解锁、置顶置底、编组或取消编组等动作。
10. 用户点击 AI 辅助入口时，系统以文件组件或文件内容作为上下文进入 AI 辅助流程。
11. 操作完成后，文件组件留在画布中，并参与协作同步、撤销重做、选择、多选和导出。

备选流程：
1. 文件仍在上传或处理中时，系统展示进行中状态，并限制依赖文件完成后的动作。
2. 非音频文件不显示音频转文本入口。
3. 多选文件和其它对象时，系统进入多选状态，只展示多选适用动作。
4. 文件组件被锁定时，系统允许查看和按策略下载，但不允许移动、缩放、重命名、删除或转写。

异常流程：
1. 文件上传或读取失败时，系统展示失败反馈，并允许用户删除或重试。
2. 文件下载失败时，系统提示下载失败，并保留原文件组件。
3. 音频转文本失败时，系统保留原音频文件组件，并提示用户重试。
4. 用户没有编辑权限时，系统不添加文件，也不允许修改已有文件组件。
5. 协作过程中文件组件被其他用户删除或锁定时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 文件组件被添加、移动、缩放、重命名、下载、转写、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 文件名、文件类型、上传/处理状态、下载状态、位置、尺寸和锁定状态都属于用户可感知状态。
4. 音频转文本只适用于当前产品识别为音频的文件组件。
