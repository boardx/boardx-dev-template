Use Case 名称：
使用文本组件

Actor：
Board Owner、Board Admin、Board Member

目标：
用户在 Board 中创建独立文本，用文本表达说明、标题或内容，并调整文本样式和位置。

系统边界：
BoardX / CanvasX Widgets

前端入口：
1. Board 主工具栏中的文本入口。
2. 画布上已有文本组件。
3. 选中文本后出现的 Widget Menu。
4. 对文本右键或打开更多菜单后的 Context Menu。

前置条件：
1. 用户已打开有编辑权限的 Board。
2. 目标组件类型在当前 Board 中可用。

触发条件：
用户选择文本工具，或选中画布上的已有文本。

主流程：
1. 用户点击文本工具并在画布目标位置点击，系统创建文本组件并显示文本光标。
2. 用户输入文字，系统在画布上显示文本内容。
3. 用户选中文本组件，系统显示选中框、缩放控制点和文本相关 Widget Menu。
4. 用户拖拽文本组件，系统移动文本位置。
5. 用户拖拽控制点，系统改变文本框宽高或排版区域。
6. 用户调整字体、字号、加粗、字体颜色、背景色、文本对齐方式和边框显示，系统更新文本外观。
7. 用户使用“转换为多个便利贴”能力时，系统把文本内容拆分为多个便利贴，并在画布上生成对应便利贴。
8. 用户通过 Context Menu 对文本执行复制、剪切、粘贴、删除、锁定、解锁、置顶置底、编组或取消编组等动作。
9. 用户点击 AI 辅助入口时，系统以当前文本内容作为上下文进入 AI 辅助流程。
10. 操作完成后，文本组件留在画布中，并参与协作同步、撤销重做、选择、多选和导出。

备选流程：
1. 用户创建空文本后离开编辑状态，系统必须给出可感知结果：保留空文本选中状态，或移除空对象并回到选择状态。
2. 多选文本组件时，系统进入多选状态，只展示多选适用动作。
3. 文本被锁定时，系统允许查看，但不允许输入、移动、缩放、改样式或删除。

异常流程：
1. 用户没有编辑权限时，系统不创建文本，也不允许修改已有文本。
2. 转换为便利贴失败时，系统保留原文本组件。
3. AI 辅助失败时，系统保留原文本内容和样式。
4. 协作过程中文本被其他用户删除或锁定时，系统关闭当前编辑状态并刷新画布。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 文本被创建、修改、移动、缩放、转换、锁定、删除，或系统明确提示当前动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 组件能力由对象类型、锁定状态、选中数量和角色权限决定。
2. Widget Menu 与 Context Menu 都不能绕过 Board 编辑权限。
3. 文本内容、字体、字号、颜色、背景、边框、对齐、尺寸和位置都属于用户可感知状态。
4. 文本转换为便利贴后，应保留可理解的内容拆分结果。
