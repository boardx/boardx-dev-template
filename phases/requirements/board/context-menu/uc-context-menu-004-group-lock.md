Use Case 名称：
编组、取消编组、锁定和解锁

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
让用户管理多个对象的组合和锁定状态。

系统边界：
BoardX / Board / Context Menu

前端入口：
1. 用户在画布空白处、单个组件或多选区域打开右键 Context Menu。
2. 空白画布可显示粘贴、选择所有、导出白板、解锁所有等画布级动作；对象菜单按对象类型显示编辑、复制、剪切、粘贴、重复、删除、层级、编组/取消编组、锁定、复制为图片/文本、导出已选区域、导出白板、打开图片、下载图片、保存到模板等适用项。
3. 锁定对象菜单会收窄，通常保留层级、复制为图片/文本、导出或打开/下载图片等允许动作。

前置条件：
1. 用户已打开白板。
2. 右键目标处于当前用户可访问范围内。

触发条件：
用户在目标位置打开 Context Menu 并选择“编组、取消编组、锁定和解锁”相关菜单项。

主流程：
1. 用户框选或多选多个对象并右键，系统显示编组、取消编组或锁定等适用动作。
2. 用户选择编组，系统把多个对象作为一个组显示，并显示组级选中框。
3. 用户移动或缩放已编组对象时，系统保持组内对象相对位置。
4. 用户右键已编组对象并选择取消编组，系统恢复单个对象选择和操作。
5. 用户选择锁定，系统让对象进入锁定状态，隐藏或禁用移动、缩放、编辑、删除等入口。
6. 如果当前菜单显示解锁动作，用户选择解锁后，系统恢复对象的普通选中框、控制点和可编辑菜单。
7. 当多选里包含锁定或无权限对象时，系统只处理允许对象，或提示部分对象不可操作。
8. 操作完成后，画布保留新的编组或锁定状态，并参与协作同步。

备选流程：
1. 没有选中对象时，系统展示画布级菜单，不展示对象级删除、层级或编辑动作。
2. 剪贴板没有可用内容时，粘贴入口不可用或提示使用系统快捷键粘贴。
3. Visitor 可看到不改变内容的查看或导出动作，编辑类动作隐藏或禁用。

异常流程：
1. 权限不足时，系统提示无权限并不改变内容。
2. 复制、粘贴、导出、下载或保存模板失败时，系统显示失败反馈。
3. 菜单打开期间目标对象被删除时，系统关闭菜单或刷新可用项。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 所选操作完成，或系统明确告知该动作不可用。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. Context Menu 由当前目标对象类型、锁定状态、选中数量和角色权限共同决定。
2. 空白画布不展示对象级动作。
3. 右键菜单不能绕过白板编辑权限、锁定状态或高风险确认。
