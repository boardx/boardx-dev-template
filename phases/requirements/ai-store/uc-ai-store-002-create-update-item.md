Use Case 名称：
创建或更新 AI Store 项目

Actor：
AI Store 创作者、Team Admin、BoardX Admin、AI 服务

目标：
用户创建或维护 Agent、工具、模板或图片工具，使其可以被自己、团队或平台用户使用。

系统边界：
BoardX / AI Store

前端入口：
1. AI Store 页面 > Create
2. AI Store 创建页 > 项目类型选择
3. Agent 创建器 > 配置表单
4. Agent 创建器 > 智能体创建助手
5. Agent 创建器 > 预览区域
6. Tool、Image Tool 或 Template 创建器 > 配置表单
7. AI Store 项目卡片 > 编辑按钮
8. AI Store 项目卡片 > 发布或提交审核按钮

前置条件：
- 用户已登录。
- 用户具备创建或编辑 AI Store 项目的权限。

触发条件：
用户点击创建 AI Store 项目，或从自己的项目卡片进入编辑。

主流程：
1. 创作者点击左侧 Creation 分组中的 Create，或从自己有权管理的资源卡片点击编辑。
2. 系统展示搜索框、Agent / AI Tool / Image Tool / Template 类型 Tab、Team Publish Status 和 System Publish Status 筛选、资源卡片列表，以及“创建”按钮。
3. 用户选择 Agent、AI Tool、Image Tool 或 Template 类型，系统刷新列表和创建按钮对应的资源类型。
4. 用户点击“创建”时，系统按当前类型打开对应创建器；从卡片点击编辑时，系统打开对应编辑器并预填当前资源信息。
5. 创建或编辑器展示该类型需要填写的名称、描述、图标、标签、提示词、工具参数、模板内容、输出格式等配置项；具体字段随类型变化。
6. 用户填写或修改配置，并可上传图标、背景图、文件或模板相关内容。
7. 系统在表单中校验必填字段和格式；校验失败时展示字段错误。
8. 用户点击保存、更新、发布或提交审核等当前可见操作。
9. 系统提交保存请求，并在成功后展示保存成功、更新成功、提交成功等反馈；失败时展示失败反馈。
10. 返回资源列表后，系统按当前类型、筛选和权限展示更新后的资源。
11. 用户进入 Authorized Agents 时，系统展示被授权管理的资源；卡片显示已授权标识，并只允许在授权范围内编辑或管理。
12. 用户进入 Shared 时，系统展示 shared 管理资源；卡片显示 shared 标识，并提供分享管理入口。

备选流程：
- A1：用户编辑已有项目，系统预填当前项目数据。
- A2：用户保存草稿但不发布，项目只在创作者管理范围可见。
- A3：用户复制已有模板创建，系统带入模板内容并允许修改。
- A4：管理员编辑团队或平台项目，系统展示审核、精选或下架相关操作。
- A5：用户在 Create 页面按团队发布状态或系统发布状态筛选，系统只更新列表，不修改资源状态。
- A6：当前列表为空且无筛选条件时，系统展示创建第一个资源的入口。

异常流程：
- E1：必填字段缺失，系统在对应字段显示错误提示。
- E2：项目名称或标识冲突，系统提示修改。
- E3：用户无权编辑该项目，系统禁止进入编辑或保存。
- E4：保存失败，系统保留表单内容并提示重试。
- E5：发布校验失败，系统提示未满足发布条件。

权限与可见性：
1. 创作者只能编辑自己创建或被授权管理的项目。
2. Team Admin 可以管理团队范围内的项目。
3. BoardX Admin 可以管理平台级项目。
4. 普通订阅用户不能编辑项目定义。
5. AI 服务不决定项目是否发布或可见，只按已保存配置执行。

后置条件：
- 成功时，AI Store 项目被创建或更新。
- 发布或提交审核后，项目进入对应可见或审核状态。

不包含：
1. 不包含 AI 执行引擎内部实现。
2. 不包含第三方模型供应商配置。
3. 不包含支付结算。

业务规则：
- 项目类型决定表单字段、校验规则和使用入口。
- 可见范围决定项目是否需要团队或平台审核。
- 发布前必须满足必填信息和权限规则。
