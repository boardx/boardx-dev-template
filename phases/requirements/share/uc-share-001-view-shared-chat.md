Use Case 名称：
查看公开分享对话

Actor：
访客、注册用户

目标：
用户通过分享链接只读查看 AVA 或 Chat 对话内容。

系统边界：
BoardX / Share

前端入口：
1. 公开分享对话链接。
2. 公开分享对话页面 > Loading chat session 状态。
3. 公开分享对话页面 > 消息列表。
4. 公开分享对话页面 > Deep Research 报告详情面板。

前置条件：
分享链接有效，且包含 chat threadId；如果需要 shareToken，链接中应携带 shareToken。

触发条件：
用户打开公开分享链接。

主流程：
1. 用户在浏览器中打开 `/chatShare/{threadId}` 分享链接，链接中可以带 shareToken。
2. 系统识别 threadId 和 shareToken，并加载公开分享数据。
3. 系统加载期间展示 Loading chat session 或加载中状态；如果 threadId 缺失，展示 Invalid chat session 和 chat ID 缺失提示。
4. 系统展示分享页头部，标题使用原聊天标题或默认分享标题；如果存在 AI Agent 描述，页头显示 AI 头像和描述。
5. 系统按时间顺序展示只读消息列表，消息使用聊天消息组件渲染，可包含用户消息、AI 回复、Markdown、代码块、图片、音频、PPTX 预览或公开附件。
6. 分享页底部展示 Shared chat session / Read only 提示。
7. 用户可以滚动查看消息；对于可渲染内容，用户可以使用消息组件本身提供的只读查看能力，例如代码块复制、PPTX 翻页预览、下载公开附件或打开公开报告。
8. 如果用户打开 Deep Research 报告，系统在右侧展示报告详情面板，并提供关闭按钮。
9. 分享页不展示聊天输入框，不允许发送新消息、上传附件、编辑或删除原聊天内容。

备选流程：
- A1：分享内容为空，系统展示聊天标题和 No messages in this chat session。
- A2：Deep Research 报告未打开时，消息列表占满页面宽度；报告打开后，消息列表和报告面板分栏显示。

异常流程：
- E1：分享不存在、threadId 缺失或 token 无效，系统展示无效会话或加载错误页面。
- E2：分享内容加载失败，系统展示 Error loading chat session 或错误边界提示。
- E3：PPTX 预览链接不可用，系统提示预览链接暂时不可用，并保留下载或重新生成分享链接的建议。

权限与可见性：
1. 拥有有效分享入口的访客、注册用户、Team Owner/Admin/Member 都只能按分享权限查看公开内容。
2. 分享页是只读访问，不允许任何角色在该页面修改原线程。
3. 登录后的 Team 角色不会因为打开分享链接获得原线程管理权。
4. 系统管理员也不能通过普通公开分享链接绕过原线程的分享范围。
5. 私有内容、团队私有上下文和未公开附件不能通过公开分享入口访问。

后置条件：
用户完成公开对话查看，原始聊天线程不被修改。

不包含：
1. 分享页是只读查看页，不编辑原始内容。
2. 分享链接生成和完整 AVA 聊天发送不是该页面的主流程。

业务规则：
1. 公开分享只暴露允许分享的内容。
2. 私有对话不可通过公开链接访问。
3. 分享页面必须在无登录状态下也能表达访问结果。
