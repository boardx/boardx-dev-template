Use Case 名称：
撤销和重做画布操作

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成撤销和重做画布操作。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到 Header 中的撤销、重做按钮，以及画布中的当前内容。
2. 用户也可以使用撤销/重做快捷键恢复或重新应用最近的画布操作。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要撤回刚刚执行的创建、移动、缩放、删除、绘制、文本编辑、连接线调整或重新应用被撤销的操作。

主流程：
1. 用户完成一次可撤销的画布操作后，系统启用 Header 中的撤销按钮。
2. 用户看到撤销按钮从不可用状态变为可点击状态。
3. 用户点击撤销按钮或按撤销快捷键，系统回退最近一次可撤销操作。
4. 系统更新画布内容、选中状态和撤销/重做按钮可用状态。
5. 用户继续点击撤销，系统按操作历史逐步回退，并在画布上显示回退后的对象状态。
6. 用户点击重做按钮或按重做快捷键，系统重新应用最近一次被撤销的操作。
7. 系统恢复对应组件的内容、位置、尺寸、连接关系或删除状态，并更新同步状态。
8. 当没有可撤销或可重做动作时，系统禁用对应按钮，避免用户触发无效操作。

备选流程：
1. 用户正在输入文本时，撤销/重做优先作用于当前文本编辑上下文，退出编辑后再作用于画布历史。
2. 多人协作时，系统只允许用户按当前协作规则回退自己可回退的操作。
3. 只读用户不能通过撤销/重做改变 Board 内容。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 画布内容回到撤销或重做后的状态，撤销/重做按钮状态与历史栈一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
