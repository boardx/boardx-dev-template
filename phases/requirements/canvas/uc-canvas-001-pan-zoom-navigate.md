Use Case 名称：
平移、缩放和导航画布

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成平移、缩放和导航画布。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到无限画布、已有组件、Header、Board Menu、缩放状态和可用的导航控件。
2. 用户可以通过鼠标滚轮、触控板、缩放控件、平移工具、空格临时平移或小地图改变画布视角。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要查看画布不同区域、放大查看细节或缩小了解全局。

主流程：
1. 用户进入 Board，系统显示当前画布视角、已有组件和缩放比例。
2. 用户滚动鼠标滚轮或使用触控板缩放，系统以当前视角为中心放大或缩小画布，并更新缩放比例。
3. 用户点击 Board Menu 的平移工具，或按住空格临时进入平移状态，系统把光标切换为可拖动画布的状态。
4. 用户拖动画布空白区域，系统移动视角但不改变任何组件内容。
5. 用户点击适应屏幕、缩小、放大或当前百分比下拉入口，系统按用户选择更新缩放级别。
6. 用户通过小地图或导航区域定位画布其它区域时，系统同步移动主画布视角。
7. 用户松开空格、切回选择工具或停止操作后，系统保留新的视角和缩放状态。

备选流程：
1. 用户处于文本输入状态时，滚轮、键盘和空格优先遵守当前输入控件的交互边界。
2. 用户使用触屏设备时，系统支持单指移动或双指缩放等移动端导航方式。
3. 只读用户仍可平移、缩放和查看画布，但不能借由导航动作修改内容。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 画布视角和缩放比例与用户最后一次导航操作一致，画布内容未被导航动作改变。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
