Use Case 名称：
在画布创建和编辑组件

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成在画布创建和编辑组件。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开可编辑 Board 后看到无限画布、Board Menu、已有组件和 Widget Menu。
2. Board Menu 当前直接入口包括选择、平移、便利贴、手绘、文本、连接线、形状、资源和模板；文件和链接从资源/更多入口进入。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要在画布上新增组件，或修改已有组件的位置、大小、文本、样式或连接关系。

主流程：
1. 用户在 Board Menu 中选择便利贴、文本、形状、连接线、手绘、资源或模板入口。
2. 系统高亮当前工具，改变光标或打开对应的子菜单、资源面板或模板面板。
3. 用户在画布点击、拖拽、输入文本、绘制线条、上传文件、粘贴链接或选择模板内容。
4. 系统在画布目标位置创建对应组件，并显示选中框、控制点或创建进度。
5. 用户选中已有组件，系统显示与组件类型匹配的 Widget Menu，例如颜色、字体、线宽、锁定、删除、AI 或下载等可用操作。
6. 用户拖动组件、拖动控制点、编辑文本或选择样式，系统即时更新组件可见状态。
7. 用户完成操作后，系统保留最终组件状态，并把可撤销动作加入撤销/重做序列。

备选流程：
1. 用户按 Esc、点击选择工具或点击空白区域时，系统退出当前创建或编辑状态。
2. 对象被锁定时，系统允许查看和复制允许的内容，但隐藏或禁用移动、缩放、删除和样式修改。
3. 只读用户看不到完整创建工具，尝试编辑时系统阻止写入并提示无权限。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 画布上新增或更新的组件与用户最后确认的内容、位置、尺寸和样式一致。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
