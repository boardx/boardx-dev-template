Use Case 名称：
画布实时协作

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成画布实时协作。

系统边界：
BoardX / Canvas

前端入口：
1. 多名用户打开同一个 Board 后，在 Header 或协作区域看到在线成员头像、同步状态和协作者光标。
2. 有编辑权限的用户可以在画布上创建、移动、编辑或删除组件；只读用户可以查看实时变化。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
多个有访问权限的用户同时打开同一个 Board，并对画布进行查看或编辑。

主流程：
1. 用户 A 打开 Board，系统显示当前画布内容、同步状态和在线成员区域。
2. 用户 B 进入同一 Board 后，用户 A 看到在线成员数量或头像变化。
3. 用户 B 移动画布光标、选中对象或编辑组件时，用户 A 看到协作者光标、选择状态或内容变化。
4. 用户 A 创建、移动、删除或修改组件后，系统在用户 A 本地立即显示结果，并把变化同步给其它在线用户。
5. 其它在线用户收到变化后，画布内容更新为一致状态。
6. 当某个用户离开 Board，系统从在线成员区域移除该用户或更新在线数量。
7. 同步正常时，用户能通过 Header 或状态提示确认内容已处于同步完成或正在同步状态。

备选流程：
1. 用户只读访问时，系统同步展示他人更新，但隐藏或禁用编辑入口。
2. 多人同时操作同一对象时，系统以当前协作规则显示最终可见状态，并刷新对象选中或锁定状态。
3. 用户选择跟随协作者时，系统把自己的视角移动到被跟随用户当前区域。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 在线用户看到一致的 Board 内容、在线成员状态和同步状态；无权限用户不能通过协作通道写入内容。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
