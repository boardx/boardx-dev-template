Use Case 名称：
复制和粘贴画布内容

Actor：
Board Owner、Board Admin、Board Member、Board Visitor、public visitor

目标：
在 Board 画布中完成复制和粘贴画布内容。

系统边界：
BoardX / Canvas

前端入口：
1. 用户打开 Board 后看到画布、已有组件、选中框和 Context Menu。
2. 用户可通过键盘复制/粘贴，或通过右键 Context Menu 的复制、剪切、粘贴、复制为图片、复制为文本入口完成内容复用。

前置条件：
1. 用户已打开目标 Board。
2. 系统已加载画布内容和当前角色。

触发条件：
用户需要复用选中的画布内容，或把剪贴板中的文本、链接、图片、表格或 Board 组件粘贴到画布。

主流程：
1. 用户选中一个或多个组件，系统显示选中框和可用菜单。
2. 用户按复制快捷键或在 Context Menu 中选择复制，系统把选中内容写入剪贴板，并保持原画布内容不变。
3. 用户选择剪切时，系统复制选中内容并从画布移除原组件；该动作可撤销。
4. 用户在画布空白区域按粘贴快捷键或选择粘贴，系统读取剪贴板内容。
5. 如果剪贴板同时包含多种可粘贴类型，系统展示粘贴类型选择，让用户选择链接、组件、表格、图片或文本。
6. 系统把选择的内容粘贴到当前视角中心或用户指定位置，并显示新组件的选中状态。
7. 用户选择复制为图片或复制为文本时，系统把选中内容转换为图片或文本并给出复制中、成功或失败反馈。

备选流程：
1. 用户正在文本输入框或文本编辑态中复制粘贴时，系统优先执行文本输入控件自身的复制粘贴。
2. 用户没有选中任何组件时，复制或剪切入口不可用，或系统提示没有可复制对象。
3. 只读用户可以复制允许查看的内容，但不能通过粘贴在 Board 中创建新组件。

异常流程：
1. 画布尚未加载完成时，系统展示加载状态并阻止误操作。
2. 网络或同步异常时，系统显示异常状态并保留本地可恢复信息。
3. 剪贴板、文件或浏览器权限不可用时，系统提示原因。

权限与可见性：
1. Board Owner：可以进入 Board、编辑内容、管理标题和内容生命周期，并可在当前 Board、Room、Team 权限允许范围内管理分享范围、删除、移动、复制等高风险操作。
2. Board Admin：可以编辑内容并使用多数协作入口；涉及可见范围、删除、移动、备份恢复等管理动作时，需要具备对应 Room/Team/Board 管理权限。
3. Board Member：可以在有编辑权限的 Board 中创建、修改、移动、复制和删除自己可操作的内容；不默认拥有权限策略和 Board 生命周期管理能力。
4. Board Visitor：可以查看允许访问的内容，使用平移、缩放、查看同步状态和在线状态等查看能力；不能创建、编辑、删除或改变分享范围。
5. Public visitor：通过公开链接进入时只能获得公开策略允许的查看能力；如需要保存、编辑、加入协作或访问非公开 Board，系统应引导登录、加入空间或提示无权限。

后置条件：
1. 剪贴板内容被更新，或画布出现粘贴后的新组件；失败时原画布内容保持不变。

不包含：
1. 不描述系统内部实现。
2. 不描述系统内部存储和同步机制。
3. 不把已存在的实现模块等同于当前界面必然显示的入口。

业务规则：
1. 画布交互必须以角色权限为边界。
2. 选中、菜单和快捷键不能绕过锁定状态。
3. 同步状态应让用户知道内容是否已经被其他协作者看见。
