Use Case 名称：
管理员管理用户

Actor：
系统管理员

目标：
管理员创建、查询、编辑或删除用户账号，维护平台用户数据。

系统边界：
BoardX / Admin Panel

前端入口：
1. Admin Panel 中的 Users 页面。
2. Users 页面中的 添加用户 / 创建用户入口。
3. 用户列表中的编辑、删除、复制 ID 和手动上分操作。

前置条件：
- 管理员已登录。
- 管理员账号拥有 admin 角色。

触发条件：
管理员进入 Admin Panel 的用户管理页面。

主流程：
1. 系统管理员进入 Users 页面，系统展示用户管理标题、管理系统用户和权限设置说明、总用户数、普通用户、管理员、已禁用统计卡片、筛选区、用户表格、分页控件和添加用户入口。
2. 管理员可以在搜索框输入邮箱关键字，按管理员或用户角色筛选，并点击查询；系统刷新列表结果，点击重置后清空筛选条件。
3. 用户表格展示用户、用户 ID、邮箱、角色、积分、加入时间和操作入口；加载中显示加载中状态，没有数据时显示暂无用户数据。
4. 管理员点击分页上一页或下一页，系统加载对应页码并保持筛选条件。
5. 管理员点击创建用户入口，系统进入创建页，管理员填写用户基础信息、邮箱、角色等字段并提交。
6. 管理员点击编辑操作，系统进入编辑页，展示该用户当前信息并允许修改。
7. 管理员打开用户操作入口，可以复制用户 ID、进入编辑、手动上分或删除用户；复制成功或失败时系统显示对应提示。
8. 管理员点击手动上分，系统打开弹窗，展示当前剩余 Credit、增加 Credit 输入框和备注输入框。
9. 管理员输入正数 Credit 和备注并提交；系统展示提交中状态，成功后提示手动上分成功并关闭弹窗。
10. 管理员删除用户时，系统展示确认对话框；确认后用户被软删除，列表状态更新。

备选流程：
- A1：管理员创建新用户。
- A2：管理员只查看用户信息。

异常流程：
- E1：非 admin 用户访问，系统拒绝。
- E2：用户不存在，系统提示无法操作。
- E3：创建或更新字段无效，系统提示修正。

权限与可见性：
1. 系统管理员可以访问后台管理入口，可以查看、创建、编辑、删除用户，并为用户手动增加 Credit。
2. 系统普通用户、Team Owner、Team Admin 和 Team Member 不因团队角色获得后台用户管理权限；访问时系统拒绝或跳转。
3. Team Owner/Admin 只能在 Team 范围管理成员，不能在 Admin Panel 管理平台用户。

后置条件：
- 成功时，用户数据按管理员操作变化。

不包含：
1. 不包含普通 Team 管理页面。
2. 不包含普通用户个人资料编辑。

业务规则：
- 用户列表分页 数量上限 最大为 50。
- 删除用户后，该用户不再作为可用账号出现在正常用户列表中。
