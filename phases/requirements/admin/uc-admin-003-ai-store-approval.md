Use Case 名称：
管理员审核 AI Store

Actor：
系统管理员

目标：
管理员查看提交到 BoardX 平台审核的 AI Store 项目，并将项目批准或撤回到待审核状态。

系统边界：
BoardX / Admin Panel

前端入口：
1. Admin Panel 中的 Store Approval / AI Store Approval 页面。
2. Admin 首页或侧边菜单中的 Store Approval 入口。

前置条件：
- 管理员已登录。
- 存在提交到平台审核的 AI Store 项目。

触发条件：
管理员进入 Admin Panel 的 AI Store Approval 页面。

主流程：
1. 系统管理员进入 AI Store Approval 页面，系统加载 BoardX Admin 资源管理界面。
2. 页面展示搜索框、结果数量、资源类型 Tab、标签筛选区和资源卡片网格；加载中展示 loading 状态，加载失败时展示错误和重试入口，没有结果时展示空状态。
3. 管理员在搜索框输入名称或描述关键字，系统按关键字刷新资源卡片；管理员清空搜索后恢复当前筛选范围。
4. 管理员点击标签，系统把标签加入筛选条件；已选标签高亮，管理员可以再次点击取消单个标签，或点击 Clear All 清除全部标签。
5. 管理员在 Agent、AI Tool、Image Tool、Template 类型间切换，系统按当前类型展示资源，并保留或重新计算搜索、标签筛选结果。
6. 资源卡片展示图标或预览图、名称、描述、标签，以及 Featured、Deep Agent、专家、共享或已授权等当前资源具备的标识。
7. 管理员点击卡片主体，系统打开资源详情弹窗或详情视图，展示资源名称、描述、公开配置内容，以及该资源提供的指令或提示词信息。
8. 只有 `isBoardXResource` 的资源卡片展示平台审核按钮；非 BoardX Resource 不显示平台审核按钮。
9. 如果资源当前 BoardX 审核状态不是 APPROVED，审核按钮显示待批准状态；管理员点击后，系统打开确认批准弹窗。
10. 如果资源当前 BoardX 审核状态是 APPROVED，审核按钮显示已批准状态；管理员点击后，系统打开撤销批准确认弹窗。
11. 确认弹窗展示当前操作含义，并展示资源名称、描述和指令/提示词内容（如果该资源提供）。
12. 管理员点击取消，系统关闭确认弹窗，不改变资源状态。
13. 管理员点击确认，系统提交审核状态更新；确认按钮显示 loading，避免重复提交。
14. 更新成功后，系统关闭确认弹窗，并把资源的 BoardX 审核状态在 APPROVED 与 PENDING 之间切换；资源卡片上的审核按钮状态同步更新。
15. 更新失败时，系统保留原审核状态，关闭或保留当前弹窗并显示失败反馈，管理员可以稍后重试。

备选流程：
- A1：管理员取消确认弹窗，系统不改变资源状态。
- A2：管理员只搜索、筛选、切换类型和查看资源详情，不执行审核动作。
- A3：当前搜索、标签或类型筛选没有匹配结果时，系统显示空状态，管理员可以清空搜索或筛选条件。
- A4：资源不是 BoardX Resource 时，管理员仍可查看卡片和详情，但不能从该卡片执行平台审核。

异常流程：
- E1：资源列表加载失败，系统展示错误状态和重试入口。
- E2：审核状态更新失败，系统保留原状态并允许管理员稍后重试。
- E3：资源在操作期间被删除、权限变化或状态已被其他管理员更新时，系统刷新列表或保留原状态并提示操作失败。

权限与可见性：
1. 系统管理员可以进入平台级 Store Approval 页面，对平台审核资源执行批准或撤销批准。
2. Team Owner/Admin 可以在 Team 范围参与团队审核，但不能替代系统管理员执行平台官方审批。
3. Team Member 或 AI Store 创建者可以提交或查看自己有权访问的项目状态，但不能在 Admin Panel 审批平台项目。
4. 非系统管理员访问后台审核页面时，系统拒绝或跳转。

后置条件：
- 成功时，资源的 BoardX 审核状态在 APPROVED 与 PENDING 之间变更。
- 失败或取消时，资源审核状态不变。

不包含：
1. 后台审批页的可确认操作是批准或撤销批准。
2. 风险评估、内容安全复核流程和 Team 管理不是该页面的主流程。
3. 不包含创建、编辑或删除 AI Store 项目。

业务规则：
- 只有系统管理员可以在 Admin Panel 执行平台级批准或撤销批准。
- 该页面展示和操作的是符合平台审核条件的 AI Store 资源。
- 只有 BoardX Resource 展示平台审核按钮。
- 审核确认弹窗必须让管理员在提交前看到资源名称、描述和指令/提示词内容。
