Use Case 名称：
在房间中新建聊天

Actor：
Room member、Room admin、Room owner

目标：
用户在房间 Chat 页签中新建一个与当前房间关联的 AVA 聊天线程。

系统边界：
BoardX / Room Chat

前端入口：
房间 Chat 页签中的 New Chat 按钮。

前置条件：
- 用户已登录。
- 用户是当前房间成员。
- 当前团队与房间信息已加载。

触发条件：
用户点击 New Chat。

主流程：
1. 用户进入房间页面并点击 Chat 页签。
2. 系统展示房间聊天列表，顶部显示 Chat List 和 New Chat 按钮。
3. 如果还没有聊天，系统展示空状态，提示新聊天创建后会出现在这里。
4. 用户点击 New Chat。
5. 系统检查是否存在当前房间下尚未发送消息的虚拟线程；如果存在，直接打开该线程。
6. 如果不存在，系统创建一个新的虚拟聊天线程，并把它设为当前聊天。
7. 页面切换到聊天工作区，中间显示 AVA 聊天区域，左侧为 Room Files，右侧为 Studio。

备选流程：
- A1：已有未使用的虚拟线程时，系统复用该线程，不创建新的线程。
- A2：用户返回房间列表，当前线程选择被清除。

异常流程：
- E1：创建线程失败时，系统不创建空白线程，并在当前聊天区域或全局提示中显示失败反馈。

权限与可见性：
- owner/admin/member 都可以在自己已加入的房间中新建房间聊天。
- 非房间成员不能进入房间 Chat 页签。
- visitor/未登录用户不能新建房间聊天。

后置条件：
- 成功时，当前聊天线程被选中。
- 成功时，用户进入房间聊天工作区。

不包含：
- 不包含 AI 回复生成的内部逻辑。
- 不包含线程持久化细节。

业务规则：
- 新建虚拟线程何时正式出现在后端线程列表，需结合聊天保存逻辑进一步确认。
