Use Case 名称：
删除房间聊天

Actor：
聊天创建者或当前用户相关线程的用户

目标：
用户从房间聊天列表删除自己可删除的聊天线程。

系统边界：
BoardX / Room Chat

前端入口：
房间 Chat 页签中聊天卡片的更多菜单。

前置条件：
- 用户已登录。
- 用户是当前房间成员。
- 房间聊天列表中存在可删除线程。

触发条件：
用户在聊天卡片更多菜单中点击 Delete。

主流程：
1. 用户进入房间 Chat 页签，看到聊天线程卡片。
2. 对于当前用户创建或归属当前用户的线程，卡片右上角显示更多菜单。
3. 用户点击更多菜单，选择 Delete。
4. 系统打开确认弹窗，展示删除标题和确认说明；如果线程有名称，说明中包含线程名称。
5. 用户点击 Delete，系统把该线程标记为删除中，并从当前列表中隐藏。
6. 系统删除聊天线程。
7. 如果用户当前正打开该线程，系统返回房间 Chat 列表。

备选流程：
- A1：用户点击 Cancel，系统关闭弹窗，不删除线程。

异常流程：
- E1：删除失败，系统记录错误，并把线程恢复到列表中。
- E2：删除过程中，删除按钮显示加载状态并禁用重复点击。

权限与可见性：
- owner/admin/member 都只能在页面显示删除菜单的线程上发起删除。
- 当前只确认 room owner/admin 不会因为角色自动看到其他成员线程的删除菜单。
- visitor/未加入房间的用户不能看到房间聊天删除入口。

后置条件：
- 删除成功时，线程不再显示在当前房间聊天列表中。
- 如果正在查看被删除线程，用户回到房间 Chat 列表。

不包含：
- 不包含恢复已删除聊天。
- 不包含批量删除。

业务规则：
- 删除入口只对当前用户可删除的线程展示；提交删除后如权限被拒绝，系统保留线程并提示无权限。
