Use Case 名称：
查看并打开房间聊天

Actor：
Room member、Room admin、Room owner

目标：
用户查看当前房间的聊天线程列表，并打开某个聊天继续协作。

系统边界：
BoardX / Room Chat

前端入口：
房间 Chat 页签中的聊天列表。

前置条件：
- 用户已登录。
- 用户是当前房间成员。

触发条件：
用户进入 Chat 页签，或点击某个聊天卡片。

主流程：
1. 用户进入房间 Chat 页签。
2. 系统加载当前团队、当前房间下的聊天线程，并合并本地已存在的房间线程。
3. 系统过滤掉虚拟线程和已被当前页面标记删除的线程。
4. 系统按最近更新时间或创建时间倒序排列线程。
5. 聊天按日期分组展示，例如今天、昨天、几天前或具体日期。
6. 每个聊天卡片展示名称、日期、最后编辑者或创建者信息。
7. 用户点击某个聊天卡片。
8. 系统把该线程设为当前聊天，并进入房间聊天详情地址。
9. 页面显示返回房间按钮、AI agent 选择入口、聊天头部菜单、AVA 聊天主体、Room Files 和 Studio 面板。
10. 如果打开的是其他成员创建的已保存聊天线程，聊天输入和文件操作区域显示仅查看状态。

备选流程：
- A1：没有线程时，系统展示空状态，并提供 New Chat。
- A2：用户点击返回房间，系统回到 Chat 列表。

异常流程：
- E1：线程加载中，系统展示骨架卡片。
- E2：正在删除的线程卡片会降低可操作性，避免重复操作。

权限与可见性：
- owner/admin/member 都可以查看当前房间可见的聊天列表。
- owner/admin/member 的房间角色不会自动授予编辑其他成员聊天线程的权限；他人创建的已保存线程以仅查看方式呈现。
- 删除菜单只在当前用户相关的线程卡片上显示。
- visitor/未加入房间的用户不能查看或打开房间聊天列表。

后置条件：
- 用户打开选中的聊天线程。
- 当前聊天状态被同步到页面。

不包含：
- 不包含跨房间聊天聚合。
- 不包含 AVA agent 的详细配置。

业务规则：
- “最后编辑者”展示当前线程可用的第一个有效名称；没有可用名称时展示默认占位。
