Use Case 名称：
在房间聊天中发送消息

Actor：
Room member、Room admin、Room owner

目标：
用户在房间聊天线程中向 AVA 发送消息，并在当前房间上下文中继续协作。

系统边界：
BoardX / Room Chat

前端入口：
房间聊天线程中的 AVA 聊天输入区。

前置条件：
- 用户已登录。
- 用户是当前房间成员。
- 用户已新建或打开一个房间聊天线程。

触发条件：
用户在聊天输入区输入内容并发送。

主流程：
1. 用户打开房间聊天线程后，系统展示三栏聊天工作区。
2. 中间聊天区域展示 AVA 聊天主体。
3. 用户可以在聊天输入区输入问题或指令。
4. 用户发送消息后，系统在当前聊天线程中追加用户消息。
5. 系统以 room chat 类型和当前 roomId 处理对话，使回复与当前房间关联。
6. 用户可以在同一线程中继续发送消息。
7. 用户点击返回房间，系统清除当前聊天选择，并回到房间 Chat 列表。

备选流程：
- A1：用户收起 Room Files 或 Studio 面板后继续发送消息。
- A2：用户更换可用 AI agent 后继续对话。

异常流程：
- E1：如果当前线程被删除，系统返回房间 Chat 列表。
- E2：AI 回复失败或额度问题由 AVA 聊天区展示；当前 Room Chat 容器只承载聊天工作区。

权限与可见性：
- owner/admin/member 都可以在自己创建或可编辑的房间聊天线程中发送消息。
- 打开其他成员创建的已保存聊天线程时，页面提示仅可查看内容，不能发送消息或修改。
- 非房间成员不能进入该线程。
- visitor/未登录用户不能发送房间聊天消息。

后置条件：
- 用户消息出现在当前线程中。
- AI 回复成功时，对话继续展示在当前线程。

不包含：
- 不包含消息编辑、复制、语音输入等 AVA 通用能力。
- 不包含模型选择与计费细节。

业务规则：
- 房间文件和 Studio 面板不作为本用例的默认消息上下文；只有当聊天区明确显示已引用文件或 Studio 内容时，消息才携带对应上下文。
