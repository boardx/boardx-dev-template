Use Case 名称：
删除知识库文件

Actor：
文件上传者、Team 管理角色、Agent/AI Tool 创建者

目标：
用户删除不再需要或错误上传的知识库文件，使其不再出现在当前知识库列表中。

系统边界：
BoardX / 知识库

前端入口：
1. Personal Knowledge Base 页面 > 文件列表。
2. Team Knowledge Base 页面 > 文件列表。
3. Agent 或 AI Tool 创建器 > Knowledge 文件列表。
4. 知识库文件行 > 删除按钮。

前置条件：
- 用户已登录。
- 用户有权管理目标文件。
- 文件存在。

触发条件：
用户在知识库文件列表中点击删除按钮。

主流程：
1. 用户进入知识库文件列表，系统展示每个文件的名称、状态、上传人、更新时间、大小和操作按钮。
2. 用户可通过搜索框过滤文件名来定位文件。
3. 用户点击文件行上的删除按钮。
4. 系统提交删除请求；删除过程中当前列表保持可见。
5. 删除成功后，系统从当前列表移除该文件，并展示 deleteFileSuccess 提示。
6. 删除失败时，系统保留文件行，并展示 deleteFileError 提示。
7. 删除后用户可以点击 Refresh 重新加载列表，确认文件不再出现。

备选流程：
- A1：用户未点击删除，系统不做变更。

异常流程：
- E1：文件不存在，系统提示删除失败。
- E2：用户无权限删除，系统拒绝或返回删除失败。
- E3：删除服务异常，系统提示稍后重试。

权限与可见性：
1. 普通用户可以删除自己有权管理的个人知识库文件。
2. Team 成员只能删除自己在当前 Team 上下文中有管理权限的文件。
3. Team Owner/Admin 可以删除 Team 范围内可管理的知识库文件。
4. Agent/AI Tool 创建者可以删除对应创建器上下文中有管理权限的知识文件。
5. 系统管理员后台权限不自动允许绕过 Team、Agent 或 Tool 的文件隔离。

后置条件：
- 成功时，文件不再出现在当前知识库列表。

不包含：
1. 不包含删除确认弹窗。
2. 不包含文件存储物理清理策略。
3. 不包含知识检索的内部处理机制。

业务规则：
- 删除后应让对应 personal、team、agent 或 tool 上下文的文件列表不再展示该文件。
- 删除失败不得从前端列表移除文件。
