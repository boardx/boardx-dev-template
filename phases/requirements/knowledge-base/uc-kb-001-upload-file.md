Use Case 名称：
上传知识库文件

Actor：
普通用户、Team 成员、Agent/AI Tool 创建者、AI 服务

目标：
用户上传文件到个人、团队、Agent 或 AI Tool 知识库，使后续 AI 能在对应上下文中使用。

系统边界：
BoardX / 知识库

前端入口：
1. Personal Knowledge Base 页面。
2. Team Knowledge Base 页面。
3. Agent 或 AI Tool 创建器中的 Knowledge 区域。
4. 知识库页面 > Upload File 按钮。
5. 上传弹窗 > 拖拽上传区域和 Select Files 按钮。

前置条件：
- 用户已登录。
- 用户拥有目标知识库范围的访问权限。

触发条件：
用户在知识库页面或创建器知识库区域选择文件。

主流程：
1. 用户进入个人或团队知识库页面，或在 Agent/AI Tool 创建器中打开知识库区域；系统根据当前上下文展示标题、说明、搜索框、Refresh 按钮、Upload File 按钮和文件列表区域。
2. 用户点击 Upload File，系统打开上传弹窗，展示弹窗标题、当前上下文说明、拖拽上传区域、支持格式说明、Select Files 按钮和关闭按钮。
3. 用户拖入文件或点击 Select Files 选择文件；系统支持多选，并在 File List 中显示已选数量和最大数量。
4. 系统校验文件扩展名是否属于 pdf、txt、md、doc、docx、json、csv、xlsx、xls 等允许类型，并校验单文件大小不超过 50MB。
5. 校验失败时，系统通过提示说明文件类型不支持、文件过大或超过最大文件数限制。
6. 校验通过后，系统自动开始上传，不要求用户再点击提交。
7. 上传列表展示文件名、大小、状态图标、进度条和状态文案；上传中显示 uploading，处理检查阶段显示 checking processing status，完成显示 completed，失败显示错误说明。
8. 系统按上下文把文件绑定到个人、团队、Agent 或 AI Tool 范围，并交给 AI 服务进行知识处理。
9. 文件处理完成或失败后，系统展示成功或失败提示；成功上传后关闭弹窗并刷新当前知识库文件列表。
10. 用户尝试在上传中关闭弹窗时，系统提示 upload in progress，并阻止关闭；上传结束后可关闭弹窗。

备选流程：
- A1：用户移除尚未开始上传或失败状态的文件。
- A2：用户选择多个文件，系统按批次并发上传。

异常流程：
- E1：文件上传失败，系统在该文件行展示 error 状态，并提示上传失败数量。
- E2：知识处理失败，系统展示 error 状态和错误说明。
- E3：自动上传过程异常，系统展示 auto upload failed 提示。

权限与可见性：
1. 普通用户可以上传到自己的个人知识库。
2. Team 成员只能在自己有权访问的 Team 上下文上传文件，不能跨 Team 上传。
3. Agent/AI Tool 创建者可以在对应创建器知识库区域上传专用知识文件。
4. 系统管理员的 Admin Panel 权限不自动扩大知识库文件的 Team 业务可见范围。
5. AI 服务只处理系统传入的当前上下文文件，不改变用户的管理权限。

后置条件：
- 成功时，文件元数据被保存，用户可在知识库列表查看文件。

不包含：
1. 不包含文件存储实现。
2. 不包含知识检索的内部处理机制。
3. 不包含 AI 模型推理过程。

业务规则：
- 文件选择后自动上传并进入知识处理。
- 文件状态包括 uploading、vectorizing、completed、error；列表中处理中的后端状态展示为 processing。
- 单文件大小上限为 50MB。
