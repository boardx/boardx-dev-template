Use Case 名称：
查看知识库文件列表

Actor：
普通用户、Team 成员、Agent/AI Tool 创建者

目标：
用户查看已上传知识库文件，并下载或打开有权访问的文件。

系统边界：
BoardX / 知识库

前端入口：
1. Personal Knowledge Base 页面 > 文件列表。
2. Team Knowledge Base 页面 > 文件列表。
3. Agent 或 AI Tool 创建器 > Knowledge 文件列表。
4. 知识库页面 > 搜索框。
5. 知识库文件行 > 下载按钮。

前置条件：
- 用户已登录。
- 用户有权访问目标知识库范围。

触发条件：
用户进入知识库页面或点击下载按钮。

主流程：
1. 用户进入个人、Team、Agent 或 AI Tool 知识库页面；系统展示对应标题、上下文说明、Refresh 按钮、Upload File 按钮、搜索框和文件列表。
2. 系统加载文件时展示骨架屏；没有文件时展示空状态，并在无搜索条件时提供上传第一个文件的入口。
3. 文件列表展示文件名称、处理状态、上传人、更新时间、向量数量（如果有）、文件大小，以及下载和删除操作。
4. 文件状态以标签展示 processing、completed、error 或 unknown。
5. 用户在搜索框输入关键词，系统按文件名或原始文件名过滤当前列表；没有匹配时展示无匹配文件提示。
6. 用户点击 Refresh，系统重新请求当前上下文的文件列表并刷新状态。
7. 如果文件超过当前页容量，用户点击 Load More，系统加载下一页并追加到列表下方。
8. 用户点击下载按钮，系统提示正在下载。
9. 系统先尝试通过受保护下载入口获取文件内容；成功时浏览器下载文件并提示下载完成。
10. 如果受保护下载失败，系统尝试直接访问文件 URL；仍失败时在新标签页打开可访问的文件 URL，并提示已打开。

备选流程：
- A1：用户只查看文件状态，不下载文件。
- A2：用户通过搜索框筛选文件后再点击下载。

异常流程：
- E1：文件没有可用 URL，系统提示文件地址不可用。
- E2：下载失败，系统提示下载失败。
- E3：文件列表加载失败，系统展示空列表或允许用户刷新重试。

权限与可见性：
1. 普通用户只能查看自己的个人知识库文件。
2. Team 成员只能查看授权范围内的 Team 知识文件。
3. Agent/AI Tool 创建者只能查看对应创建器上下文中有权访问的知识文件。
4. 系统管理员后台权限不等于业务上下文文件访问权；知识文件仍按 personal、team、agent、tool 范围隔离。

后置条件：
- 成功时，用户完成文件查看、下载或打开。

不包含：
1. 不包含文件存储实现。
2. 不包含知识检索的内部处理机制。
3. 不包含 AI 模型推理过程。

业务规则：
- 列表每次请求默认按页加载。
- processing 或 error 状态文件可以出现在列表中，但不能被描述为已经可用的知识上下文。
