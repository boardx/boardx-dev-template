Use Case 名称：
提交反馈

Actor：
注册用户

目标：
用户向 BoardX 提交问题或建议，并可附带图片帮助说明。

系统边界：
BoardX / Feedback

前端入口：
1. 页面中的反馈入口。
2. 反馈弹窗或反馈表单。

前置条件：
用户已登录并打开反馈入口。

触发条件：
用户填写反馈内容并提交。

主流程：
1. 用户点击 BoardX 页面中的反馈入口。
2. 系统打开反馈弹窗，并展示“提交反馈”卡片、反馈内容输入区、上传附件入口、“提交”和“取消”按钮。
3. 用户在输入区填写问题描述、建议内容或复现说明。
4. 用户选择是否上传附件；系统保留已选择文件，并允许在提交前移除或重新选择。
5. 用户点击提交。
6. 系统校验反馈内容不能为空；为空时展示提交错误提示，弹窗保持打开。
7. 校验通过后，提交按钮显示“提交中...”且禁用重复提交。
8. 系统把附件转换为 Base64，并随反馈内容、浏览器 user agent 和当前用户信息一起提交。
9. 提交成功后，系统清空当前白板的反馈草稿，关闭弹窗并展示“提交成功”。
10. 提交失败时，系统展示提交错误提示，用户可继续修改后重试。
11. 用户点击取消时，系统关闭弹窗，不提交当前内容。

备选流程：
- A1：用户不上传附件，仅提交文字反馈。
- A2：用户关闭反馈入口，系统保留或放弃未提交内容取决于当前页面状态。

异常流程：
- E1：反馈内容为空，系统阻止提交并提示用户补充内容。
- E2：提交失败，系统提示重试。
- E3：附件读取或提交失败，系统提示用户移除附件或稍后重试。

权限与可见性：
1. 注册用户可以提交反馈。
2. 反馈提交会带上登录用户上下文；未登录用户不作为本用例主流程，未登录状态下系统应引导登录或隐藏提交入口。

后置条件：
反馈被提交。

不包含：
1. 不包含客服处理流程。
2. 不包含附件保存和后台处理规则。

业务规则：
1. 反馈内容不能为空。
2. 上传入口接受图片附件。
