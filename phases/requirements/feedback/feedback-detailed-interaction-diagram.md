# Feedback 详细交互图

本图按 Feedback 模块当前 Use Case 的主流程展开，重点表达每个界面能看到什么、能操作什么、操作后出现什么状态。若某个 UC 继续细化，应该同步更新该 UC 的主流程和本图。

## 模块交互展开

该部分保留当前模块已有交互图，用于表达模块内主要页面区域之间的流转。

```mermaid
flowchart TD
  Entry["打开反馈入口"] --> Form["看到反馈文本框、附件入口和提交按钮"]
  Form --> Input["输入反馈内容"]
  Form --> Attach["上传附件"]
  Attach --> AttachState["附件显示上传成功或失败"]
  Input --> Submit["提交反馈"]
  Submit --> Loading["显示提交中"]
  Loading --> Success["显示提交成功并关闭或清空表单"]
  Loading --> Failed["显示失败并保留用户输入"]
```

## Use Case 主流程展开

```mermaid
flowchart TD
  subgraph D1G["提交反馈"]
    D1Start["Actor：注册用户"]
    D1Entry["可见入口/区域：页面中的反馈入口。；反馈弹窗或反馈表单。"]
    D1Start --> D1Entry
    D1S1["1. 用户点击 BoardX 页面中的反馈入口。"]
    D1Entry --> D1S1
    D1S2["2. 系统打开反馈弹窗，并展示“提交反馈”卡片、反馈内容输入区、上传附件入口、“提交”和<br/>“取消”按钮。"]
    D1S1 --> D1S2
    D1S3["3. 用户在输入区填写问题描述、建议内容或复现说明。"]
    D1S2 --> D1S3
    D1S4["4. 用户选择是否上传附件；系统保留已选择文件，并允许在提交前移除或重新选择。"]
    D1S3 --> D1S4
    D1S5["5. 用户点击提交。"]
    D1S4 --> D1S5
    D1S6["6. 系统校验反馈内容不能为空；为空时展示提交错误提示，弹窗保持打开。"]
    D1S5 --> D1S6
    D1S7["7. 校验通过后，提交按钮显示“提交中...”且禁用重复提交。"]
    D1S6 --> D1S7
    D1S8["8. 系统把附件转换为 Base64，并随反馈内容、浏览器 user agent 和当前<br/>用户信息一起提交。"]
    D1S7 --> D1S8
    D1S9["9. 提交成功后，系统清空当前白板的反馈草稿，关闭弹窗并展示“提交成功”。"]
    D1S8 --> D1S9
    D1S10["10. 提交失败时，系统展示提交错误提示，用户可继续修改后重试。"]
    D1S9 --> D1S10
    D1S11["11. 用户点击取消时，系统关闭弹窗，不提交当前内容。"]
    D1S10 --> D1S11
    D1Alt{"备选：A1：用户不上传附件，仅提交文字反馈。；A2：用户关闭反馈入口，系统保留或放弃未提交内容<br/>取决于当前页面状态。"}
    D1S11 --> D1Alt
    D1Err{"异常：E1：反馈内容为空，系统阻止提交并提示用户补充内容。；E2：提交失败，系统提示重试。；E<br/>3：附件读取或提交失败，系统提示用户移除附件或稍后重试。"}
    D1S11 --> D1Err
    D1Perm["权限/可见性：1. 注册用户可以提交反馈。；2. 反馈提交会带上登录用户上下文；未登录用户不作为本用例<br/>主流程，未登录状态下系统应引导登录或隐藏提交入口。"]
    D1S11 --> D1Perm
  end
```
