Use Case 名称：
生成 Studio 制品

Actor：
普通用户、Team 成员、AI 服务

目标：
用户通过 Studio 生成音频概览、演示文稿或信息图，并在聊天中查看、播放、下载或预览生成结果。

系统边界：
BoardX / Studio

前端入口：
1. Room 页面中的 Studio 面板。
2. Studio 面板中的音频概览、演示文稿、生成信息图入口。
3. 生成弹窗中的输入框、配置项、语音输入和 @ 文件引用。
4. 聊天结果中的音频、PPTX 或图片附件。

前置条件：
用户已进入支持 Studio 的 Room 或聊天上下文，并具备使用 AI 生成能力的权限。

触发条件：
用户选择 Studio 制品类型并点击生成。

主流程：
1. 用户打开 Studio 面板，系统展示 Studio 标题、可用工具列表和收起按钮。
2. 系统展示可用 Studio 工具：音频概览、演示文稿、生成信息图；如果当前对话为只读模式，系统展示只读提示并禁用生成操作。
3. 用户点击某个 Studio 工具，系统打开生成弹窗。
4. 音频概览入口展示通用提示词输入框，用户可输入提示词、通过 @ 引用当前 session 文件，或使用语音输入。
5. 演示文稿入口展示格式、语言、时长、样式参考图和描述输入框。
6. 信息图入口展示语言、屏幕方向、视觉风格、详细程度和描述输入框。
7. 用户点击生成后，系统关闭弹窗，在当前聊天中插入一条 Studio 请求消息，并展示生成中状态。
8. 系统向 AI 服务提交当前 Room、Team、聊天、文件和对应制品配置。
9. 音频或信息图生成中，系统在聊天中显示生成中占位；生成成功后，音频结果以音频附件展示，信息图结果以图片或图片 Markdown 展示。
10. 演示文稿生成成功后，系统在聊天中展示“演示文稿已生成”，并在返回 PPTX 时附带 PPTX 文件和预览页。
11. 生成失败时，系统展示失败消息和 Studio 错误提示，保留当前聊天上下文。

备选流程：
- A1：用户取消弹窗，系统不发起生成。
- A2：用户不输入提示词，系统对部分工具使用默认提示。
- A3：用户通过 @ 选择当前 session 文件，系统把被引用文件纳入请求。

异常流程：
- E1：输入或上下文不完整，系统展示错误提示或由 AI 服务返回失败。
- E2：额度不足，系统展示计费不足提示，并引导充值、购买 Credit 或升级。
- E3：生成失败，系统展示失败状态和错误信息。
- E4：实时语音输入网络异常，系统展示网络不稳定或实时转录暂停提示。

权限与可见性：
1. 普通用户可以在自己可见入口中使用 Studio 制品生成能力。
2. Team 成员只能在被授权的 Team、Room、聊天、文件和额度上下文中生成。
3. 只读访问者或只读聊天查看者不能在该页面生成或修改制品。
4. 系统管理员后台权限不自动授予业务 Studio 生成权。
5. AI 服务只能读取当前任务传入且用户有权访问的上下文。

后置条件：
Studio 制品或失败消息被写入当前聊天，用户可在聊天中查看结果。

不包含：
1. 不包含生成服务内部编排。
2. 不包含文件存储实现。
3. 不包含完整资产管理系统。

业务规则：
1. Studio 生成能力受 AI、文件、计费和权限策略控制。
2. 生成任务必须关联发起用户和上下文。
3. 只读聊天只允许查看，不允许触发生成。
