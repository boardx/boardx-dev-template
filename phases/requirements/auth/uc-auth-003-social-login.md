Use Case 名称：
第三方账号登录

Actor：
使用 Google、Facebook 或 Wechat 的访客或已注册用户

目标：
用户通过第三方身份快速进入 BoardX，减少邮箱密码注册和记忆成本。

系统边界：
BoardX / 账号与身份

前端入口：
1. 登录页面。
2. 注册页面。
3. Wechat 登录回调页面。

前置条件：
- 用户拥有可用的第三方账号。
- 对应第三方登录能力已启用。
- 浏览器允许跳转到第三方认证页面或弹出第三方认证窗口。

触发条件：
用户在登录或注册页面选择第三方登录入口。

主流程：
1. 用户访问 signin 或 sign-up 页面。
2. 系统展示邮箱登录或邮箱注册表单，并在能力开启时展示第三方入口。
3. 登录页可展示 Google、Facebook、Wechat；注册页可展示 Google、Facebook，且隐藏微信入口。
4. 用户点击 Google 或 Facebook 入口，系统进入对应第三方认证流程，并保留 callback 或 returnTo 上下文。
5. 用户点击微信入口时，系统在登录页切换为微信二维码登录面板，用户可返回邮箱登录。
6. 第三方认证完成后，系统处理返回结果并建立登录状态；微信回调由 signin/wechat-callback 页面处理。
7. 登录成功后，系统进入回跳目标或登录后默认页面。
8. 登录失败、二维码加载失败或第三方能力未配置时，系统展示错误或未配置提示，用户可返回并选择其它登录方式。

备选流程：
- A1：第三方返回邮箱与系统已有邮箱匹配，系统复用或关联已有用户记录。
- A2：第三方用户原记录缺少邮箱，本次登录返回邮箱时，系统补充邮箱信息。
- A3：用户从 sign-up 页面点击第三方入口，系统仍进入第三方认证处理。
- A4：用户取消第三方授权，系统返回原登录或注册页面。

异常流程：
- E1：第三方凭证无效或过期，系统提示登录失败。
- E2：第三方未返回足够身份信息，系统提示无法完成登录。
- E3：第三方账号与现有账号冲突，系统提示用户使用原登录方式或联系支持。
- E4：系统无法创建或读取用户，系统提示稍后重试。
- E5：Wechat callback 缺少必要 code 或 state，系统提示登录失败并引导返回 signin。

权限与可见性：
1. 访客可以在 signin 或 sign-up 页面看到已启用的第三方入口。
2. 未启用的第三方能力不应显示为可点击入口。
3. 已登录用户不应重复执行第三方登录。

后置条件：
- 成功时，用户处于已登录状态。
- 新用户成功时，系统生成用户记录。
- 失败时，用户保持未登录状态。

不包含：
1. 不包含 Team、Room 或 Board 业务能力。
2. 不包含第三方登录服务的内部流程。
3. 不包含密码加密、令牌刷新等技术细节。

业务规则：
- 第三方登录入口只在对应能力可用时展示。
- 第三方新用户的奖励额度、欢迎邮件和初始状态不作为本用例主流程；当前登录页面不把这些动作展示为可操作步骤。
