Use Case 名称：
通过邮箱重置密码

Actor：
忘记密码的邮箱注册用户

目标：
用户通过邮箱接收重置链接，设置新密码并恢复账号访问。

系统边界：
BoardX / 账号与身份

外部 Actor：
邮件投递服务

前端入口：
1. 登录页中的忘记密码入口。
2. 忘记密码页面。
3. 设置新密码页面。
4. 用户邮箱中的密码重置邮件。

前置条件：
- 用户已经注册邮箱账号。
- 用户可以访问注册邮箱。

触发条件：
用户在登录页点击忘记密码，提交邮箱。

主流程：
1. 用户在 signin 页面点击“忘记密码?”入口。
2. 系统打开 forgot-password 页面，展示 BoardX 标识、页面标题、Email 输入框和发送按钮。
3. 用户输入邮箱并点击发送按钮。
4. 系统在当前页面校验邮箱必填和邮箱格式；校验失败时在输入框附近展示错误。
5. 校验通过后，发送按钮进入提交状态，系统提交找回密码请求。
6. 系统确认该邮箱对应可重置密码的邮箱账号后，生成带有效期的一次性密码重置链接。
7. 系统通过邮件投递服务向用户提交的邮箱发送密码重置邮件；邮件正文包含重置链接、有效期说明和非本人操作提示。
8. 发送成功后，系统在 forgot-password 页面展示“重置链接已发送到该邮箱，请前往邮箱查看”的成功提示。
9. 用户打开自己的邮箱，看到来自 BoardX 的密码重置邮件。
10. 用户点击邮件中的重置链接，系统打开 reset-password / password-change 页面。
11. 系统展示 BoardX 标识、页面标题、New password、Password confirmation 和设置密码按钮；如果链接已过期，页面展示过期提示。
12. 用户输入新密码和确认密码并提交。
13. 系统校验新密码最小长度和两次密码一致性；校验失败时在字段附近展示错误。
14. 提交成功后，系统展示重置成功提示，并跳转到 signin 页面。
15. 如果重置链接缺失、服务端校验失败或网络异常，系统保留页面并展示错误提示，用户可重新尝试。

备选流程：
- A1：用户在 forgot-password 页面决定返回 signin，系统不发送重置邮件。
- A2：用户多次请求重置邮件，以最新可用邮件中的链接完成重置。
- A3：用户在 password-change 页面发现密码不一致，修改确认密码后重新提交。
- A4：用户没有收到邮件，可以在等待合理时间后重新提交邮箱请求新的重置邮件。

异常流程：
- E1：forgot-password 页面 Email 为空，系统在 Email 输入框附近展示错误提示。
- E2：Email 格式无效，系统在 Email 输入框附近展示错误提示。
- E3：邮箱不存在，系统提示该邮箱未注册或无法发送重置邮件。
- E4：邮件投递服务连接失败、认证失败、发件人被拒绝、收件人投递失败或发送超时时，系统应在可接受等待时间内提示“重置邮件发送失败，请稍后重试或联系管理员”，保留用户输入的邮箱，并允许用户重新点击发送；系统不应只展示 500、Request failed 或让前端代理超时。
- E5：重置链接无效或过期，password-change 页面提示链接无效。
- E6：新密码为空或少于最小长度，系统在 New password 输入框附近展示错误提示。
- E7：确认密码与新密码不一致，系统在 Password confirmation 输入框附近展示错误提示。
- E8：服务异常，系统提示重置失败，用户可稍后重试。

权限与可见性：
1. 未登录用户可以从 signin 进入 forgot-password。
2. 只有拥有邮箱账号且能访问邮箱的用户可以完成重置。
3. 重置链接不能让用户访问其它账号。

后置条件：
- 成功时，用户密码被更新。
- 成功时，旧 session 失效，需要重新登录。
- 邮件发送成功但用户尚未点击链接时，用户密码不变。
- 失败时，原密码保持不变。

不包含：
1. 不包含 Team、Room 或 Board 业务能力。
2. 不包含邮件服务内部投递流程。
3. 不包含密码加密、令牌刷新等技术细节。

业务规则：
- 忘记密码邮箱统一转为小写。
- 密码重置邮件必须发送到用户提交且已注册的邮箱。
- 系统必须使用当前环境配置的邮件投递服务发送重置邮件，并兼容该服务要求的 TLS、STARTTLS 或认证方式；只有邮件服务明确接受投递后，页面才展示发送成功。
- 忘记密码接口响应不得返回重置 token 或重置链接；用户只能从邮箱中的 BoardX 重置邮件进入设置新密码页面。
- 重置密码链接有有效期，过期后不能继续修改密码。
- 重置密码链接应为一次性使用；用户多次申请时，应以最新有效链接为准。
- 密码重置成功后会软删除该用户已有 session。
- 邮件投递失败属于可恢复业务异常，接口应在前端代理超时前返回可理解错误信息，前端应展示该信息并允许重试。
