Use Case 名称：
创建房间

Actor：
已加入当前团队的用户

目标：
用户在当前团队下创建一个房间，用于集中管理白板、房间聊天、问卷、文件和 Studio 协作内容。

系统边界：
BoardX / Room

前端入口：
Room 列表中的 New Room 加号按钮。

前置条件：
- 用户已登录。
- 当前团队已加载。

触发条件：
用户点击 New Room 加号按钮。

主流程：
1. 用户进入 Room 列表区域，看到 Recent、房间列表、搜索框以及 New Room 入口。
2. 用户点击 New Room 旁的加号按钮。
3. 系统打开 Create Room 弹窗；在移动端以抽屉形式展示。
4. 弹窗展示 Room Name 输入框和 Create Room 按钮；移动端还展示 Cancel。
5. 用户输入房间名称；系统保留输入内容。
6. 当名称少于 3 个字符时，系统在输入框下提示名称至少需要满足长度要求，并禁用或阻止创建。
7. 用户点击 Create Room 或按 Enter 提交。
8. 系统创建房间，按钮显示 creating 状态。
9. 创建成功后，系统关闭弹窗并进入新房间页面。
10. 新房间页面展示顶部页签 Board、Chat、Survey，以及成员/设置等房间操作入口。

备选流程：
- A1：移动端用户在抽屉中点击 Cancel，系统关闭抽屉，不创建房间。
- A2：用户清空输入后停留在弹窗中，等待重新输入。

异常流程：
- E1：名称为空或少于 3 个字符，系统不创建房间，并提示长度要求。
- E2：创建失败，系统提示 roomCreateFailed。

权限与可见性：
- 创建者成为新房间的 owner。
- Room owner/admin 在房间内可以看到设置入口。
- Room member 可以进入自己已加入的房间并使用房间内容入口。
- visitor/未加入当前团队的用户不能创建房间。

后置条件：
- 成功时，新房间成为当前打开的房间。
- 成功时，房间出现在当前团队的房间列表中。
- 失败时，不创建房间。

不包含：
- 不包含白板内部编辑。
- 不包含公开房间发现与加入流程。

业务规则：
- 房间名称至少 3 个字符。
- 当前创建弹窗未展示房间类型选择控件，用户不能在创建时选择 public/private。
