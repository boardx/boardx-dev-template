Use Case 名称：
管理房间文件面板

Actor：
Room member、Room admin、Room owner

目标：
用户在房间聊天工作区上传、查看、搜索、预览或删除与当前聊天线程关联的文件，辅助 AI 对话和协作。

系统边界：
BoardX / Room Files

前端入口：
1. 房间 Chat 页签中的聊天工作区左侧 Room Files 面板。
2. 文件专用面板或 Files 区域。

前置条件：
- 用户已登录并已进入目标房间。
- 用户是该房间成员。
- 用户已新建或打开一个房间聊天线程。

触发条件：
用户进入房间 Chat 页签并打开聊天线程，或在文件面板中上传、搜索、打开、删除文件。

主流程：
1. 用户进入房间页面并点击 Chat 页签。
2. 如果用户尚未打开某个聊天线程，系统展示房间聊天列表。
3. 用户新建或打开一个聊天线程后，系统进入三栏聊天工作区。
4. 左侧展示 Room Files 面板，中间展示 AVA 聊天，右侧展示 Studio 面板。
5. 文件面板展示上传入口、允许的文件类型说明、上传进度、搜索框和文件列表。
6. 当前聊天线程不是他人创建的已保存线程时，用户拖拽文件到上传区域，或点击上传区域选择文件。
7. 系统校验文件类型；支持 PDF、Word、PowerPoint、文本、CSV、图片，文件专用面板还支持 Excel。
8. 上传过程中，用户看到文件名、进度百分比和进度条。
9. 上传成功后，文件出现在列表中；上传失败或类型不支持时，该文件行显示错误信息。
10. 用户在搜索框输入文件名关键字，系统过滤当前列表；点击清除按钮后恢复列表。
11. 用户点击预览或打开入口，系统获取文件访问链接，并在弹窗、iframe 或新窗口中展示可预览文件。
12. 用户点击删除入口，系统打开确认弹窗；确认后删除文件并刷新列表。
13. 用户可以收起左侧文件面板、重新展开，或拖动分隔条调整面板宽度。

备选流程：
- A1：用户保持左侧面板收起，只使用中间聊天区。
- A2：用户调整面板宽度后继续对话。
- A3：用户只搜索或预览文件，不上传或删除。

异常流程：
- E1：如果尚未打开聊天线程，文件列表请求不执行，用户需要先新建或打开聊天。
- E2：文件类型不支持时，系统在上传项中显示 Unsupported file type。
- E3：上传失败时，系统在上传项中显示失败原因。
- E4：文件列表加载中时，系统展示加载占位。

权限与可见性：
- owner/admin/member 在进入房间 Chat 后均可看到该工作区结构。
- 打开他人创建的已保存聊天线程时，文件面板进入仅查看模式，用户可查看文件但不能上传或删除。
- 非房间成员直接访问房间会被系统拦截并返回房间列表。
- visitor/未登录用户不能进入房间文件面板。

后置条件：
- 上传成功时，文件与当前房间和当前聊天线程关联。
- 删除成功时，文件不再显示在当前文件列表中。
- 用户可以在聊天工作区中看到或隐藏 Room Files 面板。

不包含：
- 不包含文件内容如何进入 AI 上下文。
- 不包含文件转码和预览服务的内部实现。

业务规则：
- 文件列表依赖当前打开的聊天线程；未打开聊天线程时不会展示线程文件。
