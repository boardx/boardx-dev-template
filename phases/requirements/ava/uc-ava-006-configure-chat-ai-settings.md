Use Case 名称：
配置 AVA 聊天 AI 设置

Actor：
普通用户、Team 成员、Team Admin、AI 服务

目标：
用户在发送消息前选择模型、Agent、工具或 AI 参数，使回复符合当前任务需要。

系统边界：
BoardX / AVA

前端入口：
1. AVA 聊天页 > AI 设置入口
2. AVA 聊天输入区 > 模型选择器
3. AVA 聊天输入区 > Agent 选择入口
4. AVA 聊天输入区 > AI 工具入口
5. AI Store > 使用 Agent 后进入的 AVA 聊天设置

前置条件：
- 用户已登录。
- 当前团队和可用 AI 能力已加载。

触发条件：
用户打开模型、Agent、工具或设置入口。

主流程：
1. 用户进入 AVA 聊天页面。
2. 系统在聊天头部、Agent 选择区或输入区展示当前模型、Agent、AI 工具、Image Tool、研究工具和当前聊天状态。
3. 用户打开模型选择入口。
4. 系统展示可选模型列表，并标识当前模型；不可用模型是否出现或可选取决于当前 Team、额度和配置。
5. 用户选择一个模型后，系统更新当前聊天模型，并在头部或输入区体现新的模型选择；如果选择失败，保留原模型。
6. 用户打开 Agent 或 AI 工具入口。
7. 系统展示可选 Agent、AI Tool、Image Tool 或能力入口，并显示当前选择、可用状态和不可用状态。
8. 如果当前聊天已经存在消息且 Agent 切换受限，系统禁用对应入口或提示当前聊天不可切换 Agent。
9. 用户选择 Agent 后，系统把 Agent 写入当前聊天状态，聊天头部或输入区显示该 Agent，后续发送按该 Agent 处理。
10. 用户选择 AI Tool 或 Image Tool 后，系统把工具写入当前聊天状态，输入区或工具状态区域显示已选择工具。
11. 用户选择研究类型后，输入区显示当前研究模式，下一次发送按研究任务处理。
12. 用户从 AI Store 点击“使用”或“快速对话”进入 AVA 时，系统自动带入对应 Agent，并显示在当前聊天状态中。
13. 用户发送消息后，AI 服务按当前模型、Agent、工具、研究类型和可用上下文生成回复。
14. 如果设置保存或切换失败，系统展示失败提示，并保留原有设置。

备选流程：
- A1：用户没有可用模型或模型配置未加载时，系统保留当前可用模型状态或展示当前页面已有提示。
- A2：用户从 AI Store 选择 Agent 后进入 AVA，系统自动带入该 Agent。
- A3：用户在移动端通过更多菜单打开模型或工具设置。

异常流程：
- E1：模型列表加载失败，系统提示重试并保留默认设置。
- E2：用户选择的模型不可用，系统提示权限不足或额度不足。
- E3：工具配置无效，系统阻止发送并提示用户重新选择。

权限与可见性：
1. 用户只能选择自己或团队可用的模型、Agent 和工具。
2. Team Admin 可以通过团队策略影响成员可用能力；团队限制的模型不应对普通成员开放使用。
3. 分享只读访问者不能修改 AI 设置。
4. AI 服务只能执行用户当前可用的模型、Agent 和工具。

后置条件：
- 当前聊天会话记录或使用新的 AI 设置。
- 用户发送的后续消息按新的设置处理。

不包含：
1. 不包含模型供应商内部配置。
2. 不包含 AI Store 项目创建。
3. 不包含计费策略实现。

业务规则：
- 模型、Agent 和工具选择必须在发送消息前生效。
- 不可用能力必须有明确禁用或提示状态。
- AI 设置必须随团队上下文变化而刷新。
- Team 切换后，当前聊天、Agent、研究状态和 Team 相关上下文必须刷新，避免继续使用旧 Team 的配置。
- 已有消息的聊天如果限制 Agent 切换，必须在界面上体现不可切换状态。
