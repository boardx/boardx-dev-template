Use Case 名称：
使用 AVA 建议动作

Actor：
普通用户、Team 成员、AI 服务

目标：
用户通过系统推荐的问题、快捷动作或下一步建议，快速继续 AVA 对话。

系统边界：
BoardX / AVA

前端入口：
1. AVA 聊天页 > 建议动作区域
2. AVA 聊天输入区 > 快捷问题入口
3. AVA 回复下方 > 下一步建议
4. AI Store Agent 创建器 > 建议问题配置区

前置条件：
- 用户已登录。
- 当前 AVA 页面存在可用建议动作。

触发条件：
系统在空状态、消息后或 Agent 场景中展示建议动作，用户点击其中一项。

主流程：
1. 用户进入 AVA 新聊天或已有聊天页面。
2. 当当前上下文支持建议动作时，系统展示“建议操作”区域。
3. 系统可展示推荐问题或快捷动作，例如理解文件、起草邮件、创建日程、总结趋势、头脑风暴等。
4. 用户点击某个建议动作。
5. 系统把对应动作文本填入输入框，或作为待发送内容准备。
6. 用户可继续编辑该内容。
7. 用户点击发送后，系统按普通用户消息提交，并展示发送中和生成中状态。
8. AI 回复完成后，系统在聊天区展示回复；建议动作区域可根据新的上下文刷新或隐藏。

备选流程：
- A1：用户点击建议后继续手动编辑输入内容，再发送。
- A2：Agent 预设了建议问题，系统优先展示 Agent 的建议。
- A3：没有可用建议时，系统不显示建议动作区域。

异常流程：
- E1：建议动作执行失败，系统提示重试并保留输入内容。
- E2：建议动作依赖的工具不可用，系统提示不可使用。
- E3：用户无权限使用某个建议能力，系统隐藏该建议或显示受限状态。

权限与可见性：
1. 建议动作必须匹配当前用户可用的模型、Agent 和工具权限。
2. 分享只读访问者不能执行会修改线程的建议动作。
3. 团队私有 Agent 的建议只对有权限成员显示。

后置条件：
- 成功时，建议动作转换为用户输入、消息或工具请求。
- 系统根据对话结果刷新下一轮建议。

不包含：
1. 不包含建议动作生成算法。
2. 不包含 AI Store Agent 配置管理。
3. 不包含外部系统代替用户执行后续任务。

业务规则：
- 建议动作必须与当前对话上下文相关。
- 点击建议动作后，用户应能看见输入变化或执行反馈。
- 不可用建议不得诱导用户执行无权限操作。
