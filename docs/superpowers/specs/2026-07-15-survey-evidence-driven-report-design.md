# Survey Evidence-driven Report Design

## Goal

将 Survey 报告从“模拟图表 + 固定话术 + 工作台截图导出”升级为证据驱动的专业调研报告。报告中的数字、图表和结论必须能够追溯到真实答卷；没有真实数据时只展示报告结构和明确空状态。

## Root Causes

- 报告预览将图表强制标记为模拟数据，并在零答卷时根据题目数量生成虚构数值。
- 分类级图表可能把不同问题的选项合并，导致性别、年级、态度等不同口径出现在同一坐标轴。
- 洞察按最高值、最低值套用通用模板，没有绑定具体题目、有效样本和统计口径。
- 导出直接复制工作台 DOM，包含后台页面的卡片结构和交互布局，不具备正式报告的信息层级与分页规则。

## Non-negotiable Data Rules

1. 报告不得生成模拟答卷、模拟百分比或虚构样本量。
2. 零答卷时只渲染模板、数据需求和“暂无数据”，不得输出业务判断。
3. 每个图表只使用一个统计口径；默认一题一图，不把不同题目的选项拼接到同一轴。
4. 所有比例都携带分母；单选使用有效回答数，多选同时展示选择次数与答题人数口径。
5. 低样本、缺失率高或分组样本不足时降低结论强度，并在图表附近展示限制。
6. AI 只能基于服务端聚合后的证据包生成结论，不得自行补数。

## Evidence Model

服务端先把真实答卷转换为可验证的 `ReportEvidenceBundle`：

- survey: 标题、目标、状态、题目数量。
- sample: 有效答卷数、开始/结束时间、完成率、缺失率、异常答卷统计。
- questions: 每道题的题型、有效回答数、缺失数及独立聚合结果。
- distributions: 单选/下拉选项计数和占比；多选选择次数、答题人数和选择率。
- scores: 评分/NPS 的分布、均值、中位数及有效样本量。
- textThemes: 开放题主题、频次和匿名代表性原声；原声不包含身份信息。
- limitations: 零样本、低样本、缺失、偏样和不可比较项。

每条生成结论使用结构化 `EvidenceClaim`：结论、证据引用、样本口径、置信等级、限制和建议动作。缺少证据引用的 AI 输出不得进入正式报告。

## Chart Selection

- 单选/下拉：按占比排序的水平条形图，同时展示计数、占比和 `n`。
- 多选：水平条形图，明确“选择率（选择人数 / 有效答题人数）”，不把各选项占比误当作总和 100%。
- 评分/线性量表：分布图 + 均值/中位数 KPI；样本不足时只展示分布。
- NPS：推荐者、中立者、贬损者三段结构和 NPS 计算口径。
- 数值题：中位数、四分位数和范围；不默认用类别柱状图。
- 开放题：主题频次、匿名原声和未归类比例；主题归纳标记为 AI 辅助。
- 交叉分析：只有两个变量都具有足够有效样本时生成，并显示每个分组的 `n`。

## Professional Report Structure

1. 封面：报告标题、问卷名称、生成时间和报告状态。
2. 执行摘要：最多五条证据化发现、决策影响和首要行动。
3. 样本与方法：样本量、回收区间、完成率、数据清洗和限制。
4. 核心发现：跨章节最重要的真实信号，不重复章节正文。
5. 分主题分析：每章采用“结论 → 图表证据 → 解释 → 限制 → 行动”的固定阅读顺序。
6. 风险与限制：低样本、偏样、缺失、不可比较和 AI 归纳边界。
7. 行动建议：按优先级、负责人、验证指标和时间范围组织。
8. 附录：题目口径、统计定义和可追溯证据索引。

## AI Generation Logic

- Qwen 接收 `ReportEvidenceBundle`、报告模板和用户提示词，不直接接收未经整理的杂乱答卷结构。
- 系统提示明确禁止补数、合并不同口径或把相关性写成因果关系。
- AI 返回严格 JSON：执行摘要、章节结论、证据引用、限制、行动建议。
- 服务端校验所有证据引用确实存在，数值与 evidence bundle 一致；不一致的结论被丢弃并记录警告。
- 模型不可用时保留真实统计图表和方法页，只隐藏 AI 解释，不回退到固定“最高项/最低项”套话。

## Export Rendering

- PDF/Word 使用独立报告文档模型和 A4 渲染器，不复制工作台 DOM。
- 封面、目录、章节、图表、表格和附录具有明确分页规则。
- 页眉包含问卷名称，页脚包含生成时间、保密标识和页码。
- 图表包含标题、副标题、样本量、数据来源、轴定义、数值标签和必要脚注。
- 屏幕预览与导出共享同一报告文档模型，但使用不同布局组件。
- 不导出编辑按钮、AI 面板、目录栏、卡片边框或“预览维度”等内部文案。

## Error and Empty States

- 零答卷：展示“尚无真实答卷，无法生成分析结论”，允许查看报告模板。
- 低样本：展示方向性结果，但关闭强结论、细分对比和因果表达。
- 部分题目无回答：对应模块显示缺失状态，不影响其他题目生成。
- AI 失败：真实统计和图表仍可查看和导出，AI 章节显示可重试状态。
- 数据校验失败：阻止有问题的结论进入正式报告，并在生成任务中记录具体原因。

## Verification Contract

- 单元测试证明零答卷不会产生任何非零图表值或业务结论。
- 单元测试证明不同题目的选项不会进入同一图表数据集。
- 单元测试覆盖单选、多选、评分、NPS、开放题和低样本口径。
- API 测试证明 AI evidence 引用和数值经过服务端校验。
- Playwright 使用真实提交答卷生成报告，验证图表 `n`、百分比、限制和章节结构。
- 导出测试确认文档不包含“模拟数据”“预览维度”、工作台控件或虚构数字。

## Scope

本轮修改 F12 的报告证据聚合、AI 报告输入/校验、报告预览和 PDF/Word 文档渲染。问卷创建、发布、作答协议、图像生成供应商和高级统计建模不在本轮范围。
